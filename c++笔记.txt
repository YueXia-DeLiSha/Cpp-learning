4 程序流结构
C/C++支持以下最基本的三种程序运行结构：
顺序结构：程序按顺序执行，不发生跳转。
选择结构：根据判断条件是否成立，有选择的执行相应功能。
循环结构：根据判断条件是否成立，循环执行某段代码。
4.1 选择结构
4.1.1 if语句
作用执行满足条件的语句

单行格式的if语句      if(条件){}  条件为真执行大括号内容   注意事项，if条件后面不要加分号(；)
多行格式的if语句      if(条件){条件满足执行的语句}else{条件不满足执行的语句}  
多条件的if语句        if(条件1){条件1满足执行的语句}else if(条件2){条件2满足执行的语句}else .....else{都不满足的执行的语句}
还有一种格式，嵌套if语句
嵌套if语句：在if语句中嵌套使用if语句，达到更准确的条件判断

case里放的是要和表达式结果对比的值

break; //退出当前分支

//if和switch的区别
//switch缺点，表达式的结果必须是整型或者字符型，不可以是一个区间

//if优点，表达式的结果可以是任意类型，可以是区间

4.2 循环结构
4.2.1 while循环
作用：满足循环条件，执行循环语句
语法：while(循环条件) 循环语句;
解释：只要循环条件的结果为真，就执行循环语句
注意事项：循环一定要有退出的条件，否则就是死循环
rand是一个生成随机数的函数
示例：
rand()%100    //生成一个0-99的随机数
srand随机数种子函数
示例：srand((unsigned int)time(NULL));
srand 用于初始化随机数生成器。它接受一个种子值，并根据这个种子值生成一系列伪随机数。
time(NULL) 返回的是一个 time_t 类型的值，表示当前时间的时间戳。将这个时间戳作为种子值传递给 srand 函数，可以确保每次运行程序时，随机数生成器都会从一个不同的起始点开始，从而生成不同的随机数序列。
(unsigned int) 是一个类型转换操作，将 time_t 类型的值转换为 unsigned int 类型。这种转换通常是为了确保传递给 srand 函数的值是无符号整数。
srand 用于初始化随机数生成器。它接受一个种子值，并根据这个种子值生成一系列伪随机数。
4.2.2 do...while循环
作用：满足循环条件，执行循环语句
语法：do{循环语句} while(循环条件);
注意：与while循环不同的是，do...while循环会先执行循环语句，再判断循环条件，只要循环条件的结果为真，就会重复执行循环语句。
解释：do...while循环会先执行循环语句，再判断循环条件，只要循环条件的结果为真，就会重复执行循环语句。

获取个位数：num % 10 取余 对数字取模于10就可以拿到个位数
获取十位数：num / 10 % 10 整除除以10再取余就可以拿到十位数
获取百位数：num / 100  整除除以100就可以拿到百位数
4.2.4 for循环
作用：满足循环条件，执行循环语句
语法：for(起始表达式; 循环条件; 循环后操作)
循环语句;
解释：起始表达式先执行一次，然后判断循环条件，如果条件为真，执行循环语句，然后执行循环后操作，再判断循环条件，如果条件为真，继续执行循环语句，直到循环条件为假，退出循环。
for循环的执行顺序：
1. 执行起始表达式
2. 判断循环条件，如果为真，执行循环语句，然后执行循环后操作，再判断循环条件，如果为真，继续执行循环语句，直到循环条件为假，退出循环。
3. 如果循环条件为假，直接退出循环。
4.2.5 嵌套循环
作用：在循环中嵌套使用循环，可以完成一些复杂的重复性操作
语法：外层循环控制行数，内层循环控制列数
示例：
//外层循环
for (int i = 0; i < 10; i++) {
	//内层循环
	for (int j = 0; j < 10; j++) {
		cout << "* ";
	}
	cout << endl;
}

4.3.3 goto语句
作用：可以无条件跳转到程序中指定的行
语法：goto 标记;
解释：goto语句可以无条件地跳转到程序中指定的行，标记是一个标识符，后面跟一个冒号。 
示例：
goto FLAG;
cout << "1" << endl;
cout << "2" << endl;
cout << "3" << endl;
FLAG:

注意事项：

goto语句通常用于处理特殊情况，例如错误处理或跳出多层循环。
goto语句的跳转范围是整个函数，因此需要谨慎使用，以避免跳转到未定义的代码行。
5 数组
5.1 数组定义
作用：用来存储一组相同类型的数据
语法：数据类型 数组名[元素个数];
特点1：数组中存放的数据类型必须统一
特点2：数组是由连续的内存地址组成的
特点3：数组名是数组的标识符，数组名表示整个数组，数组名中存储的是数组首元素的地址
特点4：数组中下标是从0开始的
5.2.1 一维数组
一维数组定义方式
方式1：数据类型 数组名[数组长度];
方式2：数据类型 数组名[数组长度] = {值1，值2...};
方式3：数据类型 数组名[] = {值1，值2...};
5.2.2 一维数组数组名
一维数组名称的用途：
1. 可以统计整个数组在内存中的长度
2. 可以获取数组中某个元素的地址
示例：

5.2.3 冒泡排序
作用：最常用的排序算法，对数组内元素进行排序
1. 比较相邻的元素。如果第一个比第二个大，就交换他们两个。
2.对每一对相邻元素做同样的工作，执行完毕后找到最大的元素，它会被放到数组的末尾。
3.重复以上步骤，每次比较次数-1，直到不需要比较。
sizeof(arr)/sizeof(arr[0])是计算数组的长度

for (int j =0; j < sizeof(arr)/sizeof(arr[0]) -1 - i; j++)
这里，内层循环的次数是数组长度-1再减去当前轮数i。
因为每一轮都会将当前未排序部分的最大值放到最后，所以下一轮就可以少比较一次。

比较相邻元素
从数组开头开始，逐一比较相邻的两个元素（例如 arr[0]和arr[1]）。
交换位置
如果前一个元素比后一个大（arr[j] > arr[j+1]），就交换它们的位置。
传递最大值
每一轮完整的遍历后，当前未排序部分的最大值一定会被推到末尾（就像气泡浮到水面）。
缩小范围
下一轮遍历时，忽略已排序好的末尾部分（比较次数减一），重复上述过程直到全部有序。
示例：
int arr[10] = {4,2,8,0,5,7,1,3,9,6};
// 外层循环：控制轮数（n-1轮）
for (int i = 0; i < sizeof(arr)/sizeof(arr[0]) - 1; i++) {
    // 内层循环：相邻元素两两比较（每轮减少一次比较）
    for (int j = 0; j < sizeof(arr)/sizeof(arr[0]) - 1 - i; j++) {
        if (arr[j] > arr[j + 1]) {  // 前>后？交换！
            int temp = arr[j];
            arr[j] = arr[j + 1];
            arr[j + 1] = temp;
        }
    }
}
for (int i = 0; i < sizeof(arr) / sizeof(arr[0]); i++) {
	cout << arr[i] << endl;
}

6.0 函数

6.1 概述
作用：将一段重复使用的代码封装起来，减少重复代码
一个较大的程序，一般分为若干个程序模块，每个模块实现特定的功能。
6.2 函数的定义
函数的定义一般包含五个部分：返回值类型、函数名、参数列表、函数体、返回语句。
返回值类型：指定函数返回值的类型，可以是任何数据类型，包括基本数据类型和复合数据类型。
函数名：用于标识函数，函数名应该具有描述性，以便于理解函数的功能。
参数列表：用于传递给函数的参数，参数列表可以包含零个或多个参数，每个参数由参数类型和参数名组成，参数之间用逗号分隔。
函数体：用于实现函数的具体功能，函数体可以包含任意数量的语句，包括变量声明、条件语句、循环语句等。
返回语句：用于返回函数的结果，返回语句可以包含一个表达式，表达式的值将作为函数的返回值。
语法：返回值类型 函数名(参数列表) { 函数体; 返回语句; }
6.3 函数的调用
函数的调用是指通过函数名和参数来执行函数中的代码。
函数的调用方式：
函数名(参数列表);
函数的调用时机：
当需要执行函数中的代码时，可以通过函数名和参数来调用函数。
函数的调用过程：
当调用函数时，程序会跳转到函数的定义处，执行函数体中的代码，然后返回到调用函数的地方，继续执行后续的代码。
6.4 值传递
值传递是指在调用函数时，将实际参数的值复制一份传递给函数的形式参数，函数内部对形式参数的修改不会影响到实际参数的值。
当不需要返回值时可以使用void 可以省略return
当我们做值传递时，函数形参发生改变不会影响到实参
6.5 常见形式
常见的函数定义形式有：
无参无返：函数没有参数，也没有返回值
示例：void test() { }
调用：test();
有参无返：函数有参数，没有返回值
示例：void test(int a) { }
调用：test(10);
无参有返：函数没有参数，有返回值
示例：int test() { return 10; }
调用：int num = test();
有参有返：函数有参数，有返回值
示例：int test(int a) { return a; }
调用：int num = test(10);
无类型void无法创建创建变量但可以创建函数因为无法分配内存
6.6 函数的声明
作用：告诉编译器函数名称以及如何调用函数，函数的实际主体可以单独定义。
函数声明语法：返回值类型 函数名(参数列表);
函数声明的作用：
1. 告诉编译器函数的名称和参数列表，以便于编译器进行语法检查。
2. 告诉编译器函数的返回值类型，以便于编译器进行类型检查。
声明可以有多次但是函数的定义只能有一次
6.7函数的分文件编写
作用：让代码结构更加清晰
函数分文件编写一般有4个步骤：
1. 创建后缀名为.h的头文件，用于声明函数。语法：函数返回值类型 函数名(参数列表);
2. 创建后缀名为.cpp的源文件，用于定义函数。语法：函数返回值类型 函数名(参数列表) { 函数体; }
3. 在需要调用函数的源文件中包含头文件。
4. 在需要调用函数的源文件中调用函数。
调用方法：#include"头文件名" 在源文件中也要包含头文件
在源文件中包含头文件是为了告诉编译器函数的声明，以便于编译器进行语法检查。
7 指针
7.1 指针的基本概念
指针是一个变量，其值为另一个变量的地址，即，指针变量的值存放的是另一个变量的地址。
指针变量：指针变量是一种特殊的变量，其值是一个内存地址。
指针常量：指针常量是一个固定的内存地址，不能被修改。
指针数组：指针数组是一个数组，其元素是指针变量。
指针占用大小为4或8个字节，在32位系统中，指针占用4个字节，在64位系统中，指针占用8个字节。
作用：可以通过指针间接访问内存中的数据，实现动态内存分配和内存管理。
内存编号一般从0开始记录，一般用16进制数字表示
可以用指针变量保存地址
7.2 指针的定义和使用
指针的定义语法：数据类型 *指针变量名;
指针的使用语法：指针变量名 = &变量名; 
指针的访问语法：*指针变量名 = 变量值;
指针的解引用语法：变量值 = *指针变量名;
指针的->访问语法：变量值 = 指针变量名->成员名;
指针的[]访问语法：变量值 = 指针变量名[下标];
指针的++和--操作：指针变量名++; 指针变量名--;
指针的加减操作：指针变量名 += n; 指针变量名 -= n;
(int *)表示指向int类型的指针包含两个信息：
1. 指针的类型：int *表示指向int类型的指针。
2. 指针的大小：int *表示指针的大小为4个字节。
示例：
int num =10;
int *p = &num; //指针p指向num的地址
void *voidp = &num; //void *表示指向任意类型的指针，voidp指向num的地址
int *intp = (int*)voidp; //将void *类型的指针转换为int *类型的指针

指针前面加*表示解引用，即获取指针所指向的内存地址中的数据。
7.3 指针所占内存空间
指针类型决定了指针所指向的内存空间的大小。
指针类型	占用内存空间
char	1个字节
short	2个字节
int	4个字节
long	4个字节
long long	8个字节
float	4个字节
double	8个字节
指针类型决定了指针所指向的内存空间的大小，不同类型的指针所占用的内存空间大小不同。
7.4 空指针和野指针
空指针：指针变量指向内存中编号为0的空间 作用：初始化指针变量 空指针指向的内存不可访问 0~255之间的内存编号是系统内存编号，因此不能访问
野指针：指针变量指向的内存空间地址不合法
空指针和野指针的区别：
空指针指向的内存空间是合法的，但是该内存空间没有分配任何数据。
野指针指向的内存空间是非法的，该内存空间可能已经被分配给其他数据，或者该内存空间不存在。
空指针和野指针的危害：
空指针和野指针都可能导致程序崩溃，因为它们指向的内存空间可能已经被其他数据占用，或者该内存空间不存在。
7.5 const修饰指针
const修饰指针有三种情况：
const修饰常量 示例：
int const *p = &a; //指针指向的值不能修改
*p = 100; //错误，不能通过指针修改指向的值
p = &b; //正确，指针可以指向其他变量
int * const p = &a; //指针指向的地址不能修改
p = &b; //错误，不能通过指针修改指向的地址

解释：指针指向的值为常量不可修改，可以改变指针指向的地址
const修饰指针  示例：
int a = 10;
int b = 20;
int * const p = &a; //指针指向的值不能修改
*p = 100; //正确，可以通过指针修改指向的值
p = &b; //错误，不能通过指针修改指向的地址
解释: 指针指向的地址不可修改，可以改变指针指向的值

const修饰指针常量----修饰指针指向的地址和值  示例：
int a = 10;
int b = 20;
const int * const p = &a; //指针指向的地址和值都不能修改
*p = 100; //错误，不能通过指针修改指向的值
p = &b; //错误，不能通过指针修改指向的地址
解释：指针指向的地址和值都不能修改
7.6 指针和数组
利用指针访问数组元素
数组名在大多数情况下可以当作指针来使用，因此可以通过指针来访问数组元素。可以通过指针变量名来访问数组元素。指针变量名[下标] 也可以通过指针变量名加减来访问数组元素。指针变量名++ 指针变量名-- 也可以通过指针变量名加减来访问数组元素。
示例：

cout << *p << endl; //输出数组的第二个元素
指针访问二维数组
二维数组名在大多数情况下可以当作指向一维数组的指针来使用，因此可以通过指针来访问二维数组元素。

cout << *(*p + 1) << endl; //输出二维数组的第二个元素
p++; //指针指向二维数组的第二个元素
cout << *(*p + 2) << endl; //输出二维数组的第三个元素
7.7 指针与函数

结构体可以作为函数的参数传递给函数。
传递方式有两种：值传递和指针传递。
值传递：将结构体变量的值复制一份传递给函数，函数内部对结构体变量的修改不会影响到实参的值。
指针传递：将结构体变量的地址传递给函数，函数内部通过指针访问结构体变量，可以修改实参的值。
示例：#include <iostream>

struct liang {
	int a;
    int b;
	float c;

}; //自定义一个结构体

void print(liang l) {
	std::cout << l.a << std::endl;
	std::cout << l.b << std::endl;
	std::cout << l.c << std::endl;
}

void print2(liang *l) {
	std::cout << l->a << std::endl;
	std::cout << l->b << std::endl;
	std::cout << l->c << std::endl;
}

int main() {

 liang a = {1,2,3.5}; //创建一个结构体变量
 print(a); //值传递
 
 print2(&a); //指针传递

    system("pause");
	return 0;
}

8.7 结构体中const的使用场景
作用：用const防止误操作修改结构体成员的值 语法：const 数据类型 成员名;
示例：
#include <iostream>

struct liang {
	int a;
	int b;
	const float c; //用const修饰结构体成员，防止误操作修改结构体成员的值

}; //自定义一个结构体

void print(const liang *l) {
    l->a = 100; //不可以修改，因为指针指向的结构体成员是const
}
int main() {

	liang a = { 1,2,3.5 }; //创建一个结构体变量
	print(&a); //指针传递

	system("pause");
	return 0;
}

C++核心编程
本阶段主要针对C++面向对象编程技术做详细讲解，探讨C++中的核心和精髓
1 内存分区模型
C++在执行程序时，将内存大方向划分为四个部分：
代码区：存放函数体的二进制代码，由操作系统进行管理的
全局区：存放全局变量和静态变量以及常量，程序结束后由操作系统释放
栈区：由编译器自动分配释放，存放函数的参数值，局部变量等
堆区：由程序员分配和释放，若程序员不释放，程序结束时由操作系统回收
内存四区意义：
不同区域存放的数据，赋予不同的生命周期，给我们更大的灵活编程
1.1 程序运行前
在程序编译后，生成了exe可执行程序，未执行该程序前分为两个区域
代码区：
存放CPU执行的机器指令
代码区是唯二的两个保存程序数据的内存区域之一
代码区是共享的，共享的目的是对于频繁执行的程序，只需要在内存中有一份代码即可，节省内存空间
代码区是只读的，防止程序意外地修改了机器指令

全局区：存放全局变量和静态变量以及常量，程序结束后由操作系统释放
在函数体中为局部变量分配空间，在栈区分配
在函数体定义外为全局变量分配空间，在全局/静态存储区分配
全局区是唯二的两个保存程序数据的内存区域之一
静态变量：在变量前添加关键字static，称为静态变量
静态变量在程序结束后由操作系统释放
常量：
字符串常量：以空字符结尾的字符数组，被双引号括起来的内容就算是字符串常量
const修饰的变量：coust修饰的全局变量（全局常量），coust修饰的局部变量（局部常量），局部变量不存放在全局区
常量在程序结束后由操作系统释放

1.2程序运行后
栈区：
由编译器自动分配释放，存放函数的参数值，局部变量等
注意事项：
不要返回局部变量的地址，栈区开辟的数据由编译器自动释放

堆区：
由程序员分配释放，若程序员不释放，程序结束时由操作系统回收，
在C++中主要使用new在堆区开辟内存
语法：new 数据类型(初始值)，返回的是16进制内存地址。
示例：
指针本质也是局部变量放在栈区上，指针保存的数据放在堆区

1.3 new操作符
C++利用new操作符在堆区开辟数据
堆区开辟的数据，由程序员手动开辟，手动释放，释放使用操作符delete
语法：new 数据类型 示例 int * p = new int(10);

delete 指针变量名;

2 引用

引用是C++中一种重要的类型，它为变量提供了一个别名。通过引用，可以使用另一个名字来访问同一个变量。
定义语法：数据类型 &引用名 = 原变量名;
引用需要引用一块合法的内存空间
使用语法：引用名.成员名;
示例：
#include <iostream>

	int &b = a; // b是a的引用
	b = 20; // 修改b的值，也会修改a的值
	std::cout << a << std::endl; // 输出20
	return 0;
}

2.2 引用的注意事项
引用必须初始化，引用初始化后不能改变
引用作为函数参数，可以简化代码，引用作为函数参数传递时，不会产生副本，可以修改实参的值
引用作为函数的返回值，不要返回局部变量的引用，因为局部变量在函数结束后会被销毁
引用的本质：引用的本质是给变量取了一个别名
错误语法：int &b; // 引用必须初始化
示例：

	//int &b = c; // 引用初始化后不能改变,报错
	b = c; //这是赋值操作，不是改变引用
	return 0;
}

3 函数提高
3.1 函数默认参数
在c++中，函数的形参列表中的形参是可以有默认值的，这些默认值称为函数的默认参数。
语法：返回值类型 函数名 (数据类型 形参名 = 默认值) { 函数体; }
传入数据优先级大于默认值，如果传入数据，则使用传入数据，如果未传入数据，则使用默认值
注意事项：
1.如果某个位置的形参有默认值，那么该位置往后，从左到右的所有形参都必须有默认值
2.如果函数声明中有了默认参数，那么函数实现中就不能再出现默认参数，声明和实现只能一个有默认参数

3.2 函数的占位参数
c++中的形参列表里可以有占位参数，用于占位，调用函数时必须填补该位置
语法：返回值类型 函数名 (数据类型 形参名 , 数据类型) { 函数体; }//单一的数据类型就属于占位参数
占位参数还可以有默认参数，语法：返回值类型 函数名 (数据类型 形参名, 数据类型 = 默认值) { 函数体; }

3.3 函数重载
3.3.1 函数重载概述
作用：函数名可以相同，提高复用性
函数重载满足条件：
1.同一个作用域下
2.函数名相同
3.函数参数类型不同，或者参数个数不同，或者参数顺序不同
注意：函数返回值不可以作为函数重载的条件
示例：
#include <iostream>

// 函数重载
void func() {
	std::cout << "func()调用" << std::endl;
}

void func(int a) {
	std::cout << "func(int a)调用" << std::endl;
}

void func(double a, int b) {
	std::cout << "func(double a, int b)调用" << std::endl;
}
void func(int a, double b) {
	std::cout << "func(int a, double b)调用" << std::endl;
}

3.3.2 函数重载注意事项
注意事项：
1.函数重载与函数返回值无关
2.函数重载与函数参数默认值有关
3.函数重载与函数参数类型转换有关
引用作为函数重载的条件
函数重载遇到默认参数
引用作为函数重载的条件的注意事项：
引用作为函数重载的条件时，引用类型必须不同
示例：
void func(int &a) {
	std::cout << "func(int &a)调用" << std::endl;
}

void func(const int &a) {
	std::cout << "func(const int &a)调用" << std::endl;
}

int main() {
	int a = 10;
	func(a); // 调用func(int &a)
	const int b = 20;
	func(b); // 调用func(const int &a)
	return 0;
}
函数重载遇到默认参数的注意事项：
当函数重载遇到默认参数时，可能会导致调用函数时出现二义性
示例：
void func(int a, double b = 3.14) {
	std::cout << "func(int a, double b = 3.14)调用" << std::endl;
}

void func(int a) {
    std::cout << "func(int a)调用" << std::endl;

}

int main() {
	func(10); // 二义性，编译器不知道调用哪个函数
	return 0;
}

4.类和对象
C++面向对象的三大特性：封装、继承、多态
C++认为万事万物皆对象，对象是类的实例化，对象上有其的属性和行为

类是对一类事物的抽象，是对具有相同属性和行为的事物的描述
例如：
人可以作为对象，人有属性：姓名、年龄、性别等
人有行为：吃饭、睡觉、工作等
车可以作为对象，车有属性：品牌、颜色、价格等
车有行为：启动、行驶、停车等
具有相同属性的对象可以归为一类事物，这类事物就可以用类来描述，人属于人类，车属于汽车类
4.1 封装
封装是面向对象的三大特性之一，是将数据和操作数据的函数绑定在一起，形成一个整体，对外隐藏实现细节，只暴露接口。
封装的作用：
1. 提高代码的复用性
2. 提高代码的可维护性
3. 提高代码的安全性
4.1.1 封装的意义
封装是面向对象的三大特性之一
封装的意义：
将属性和行为绑定在一起，形成一个整体，表现生活的事物
将属性和行为加以权限控制
封装意义一：
在设计类的时候，属性和行为写在一起，表现事物
语法：class 类名 {访问权限: 属性 / 行为 }；
行为一般使用函数来表示
通过类创建对象语法：类名 对象名;
对象赋值语法：对象名.属性名 = 值; 对象名.行为名();
实例化的意思：通过类创建对象的过程称为实例化，创建对象的过程就是类的实例化过程
专业术语解释:类中属性和行为统一称为成员，属性称为成员属性和成员变量，行为称为成员函数和成员方法
示例一：设计一个圆类，求圆的周长
class Circle {
public: //访问权限
	double radius; //属性 半径
	double getCircumference() { //行为 求周长
		return 2 * 3.14 * radius;
	}
};

int main() {
	Circle c;
	c.radius = 10;
	std::cout << "圆的周长为：" << c.getCircumference() << std::endl;
	return 0;
}
封装意义二：
将属性和行为加以权限控制
访问权限有三种：
public 公共权限 ：公共权限的成员可以在类内或者类外直接访问
protected 保护权限 : 保护权限的成员只能在类内和子类中访问 类外无法直接访问
private 私有权限 : 私有权限的成员只能在类内访问，类外无法直接访问 子类不可以直接访问
默认权限：类中成员的默认权限是私有权限
类内是指在类的成员函数中访问
子类是指通过继承方式创建的类
示例：
class Person {
public:
	std::string name; // 公共权限
protected:
	int age; // 保护权限
private:
	std::string password; // 私有权限
public:
	void setPassword() { // 公共权限
		name = "张三";
		age = 18;
		//password = "123456"; // 私有权限，可以在类内访问
	}
};

int main() {
	Person p;
	p.name = "张三"; // 公共权限
	//p.age = 18; // 保护权限，类外无法访问
	//p.password = "123456"; // 私有权限，类外无法访问
	return 0;
}

4.1.2 封装-C++中struct和class的区别
struct和class的区别：
struct默认权限是公共权限 public，class默认权限是私有权限 private
struct主要用于描述数据结构，class主要用于描述具有属性和行为的事物
struct也支持权限控制，可以使用public、protected、private来修饰成员

4.1.3将成员属性设置为私有
优点一：保护成员属性，防止误操作，将所有成员属性设置为私有，可以自己控制读写权限
优点二：对于写权限，我们可以检测数据的有效性，保证数据的有效性
示例代码：
class Person {
private:
	int age; // 私有权限
	int height; // 私有权限只写
	int weight = 80; // 私有权限只读
public:
	void setAge(int a) { // 写权限
		if (a < 0 || a > 150) {
			std::cout << "年龄输入有误！" << std::endl;
			return;
		}
		// 数据有效性检测，避免输入无效数据
		age = a;
	}

	int getAge() { // 读权限
		return age;
	}
	void setHeight(int h) { // 写权限
		if (h < 0 || h > 300) {
			std::cout << "身高输入有误！" << std::endl;
			return;
		}
		// 数据有效性检测，避免输入无效数据
		height = h;
	}
	int getWeight() { // 读权限
		return weight;
	}
};

4.2 对象的初始化和清理
在我们生活中电子产品大多数都有出厂设置，在某一天我们不用了需要删除隐私数据可以恢复出厂设置
C++中类的对象也有类似的机制，称为构造函数和析构函数

4.2.1 构造函数和析构函数
1. 什么时候需要自定义构造函数？
如果编译器生成的默认构造函数不能满足需求，你就需要自己写构造函数。常见情况包括：
初始化成员变量：尤其是常量成员、引用成员，必须在构造函数初始化列表中初始化。
动态资源分配：例如 new 一块内存，需要在构造函数里分配。
参数合法性检查：比如构造函数接收一个年龄，需要确保年龄在合理范围内。
与外部资源交互：如连接数据库、打开文件。
class Person {
private:
    std::string name;
    int age;
public:
    // 自定义构造函数：初始化成员，并检查参数
    Person(const std::string& n, int a) : name(n), age(a) {
        if (age < 0) throw std::invalid_argument("age must be positive");
    }
};
如果没有任何构造函数，编译器会生成一个无参的默认构造函数（仅对非内置类型成员调用其默认构造）。但一旦你定义了其他构造函数，编译器就不会自动生成默认构造函数，此时若需要默认构造，要显式定义（=default 或自己写）。

2. 什么时候需要自定义析构函数？
当对象持有 必须手动释放的资源 时，就需要自定义析构函数。例如：

用 new 分配的内存（需要 delete）

打开的文件描述符（需要 fclose）

互斥锁（需要解锁）

网络连接（需要关闭）

如果不自定义析构函数，编译器会生成一个什么都不做的析构函数，导致资源泄漏。
class String {
private:
    char* data;
public:
    String(const char* str) {
        data = new char[strlen(str) + 1];
        strcpy(data, str);
    }
    ~String() {           // 必须自定义析构函数释放内存
        delete[] data;
    }
};
3. 三大定律（Rule of Three/Five）
如果类需要自定义析构函数，通常也意味着需要自定义 拷贝构造函数 和 拷贝赋值运算符，否则默认的浅拷贝会导致多个对象指向同一块资源，造成重复释放或悬挂指针。在 C++11 后，还有移动构造函数和移动赋值运算符（Rule of Five）。
class String {
    // ... 构造、析构
    String(const String& other) {               // 拷贝构造：深拷贝
        data = new char[strlen(other.data) + 1];
        strcpy(data, other.data);
    }
    String& operator=(const String& other) {    // 拷贝赋值：深拷贝 + 自赋值保护
        if (this != &other) {
            delete[] data;
            data = new char[strlen(other.data) + 1];
            strcpy(data, other.data);
        }
        return *this;
    }
    // 移动构造/赋值（可选）
};

构造函数：当对象需要初始化（尤其是涉及资源获取、参数校验）时，显式定义构造函数。
析构函数：当对象管理非内存资源（如文件、锁）或动态内存时，必须定义析构函数来释放资源。
若定义了析构函数，通常也要定义拷贝构造和拷贝赋值（或将其禁用），以防止资源管理错误。
简而言之，构造函数负责“获取资源并建立不变式”，析构函数负责“释放资源并清理”。如果你的类不管理任何资源（例如只包含 int、double 等简单类型），编译器生成的版本就足够用了。


对象的初始化和清理是两个很重要的安全问题
一个对象或者变量没有初始状态，很可能会导致程序出现不可预知的错误
同样使用完一个对象和变量没有及时清理，也会导致一定的安全问题。

C++中提供了两种特殊的成员函数来解决对象的初始化和清理问题,这两个函数会被编译器自动调用，完成对象初始化和清理工作。
对象初始化和清理工作是编译器强制要我们做的事情，如果我们不提供这两个函数，编译器会自动提供默认的构造函数和析构函数默认为空实现。

构造函数：用于对象的初始化，在对象创建时自动调用
析构函数：用于对象的清理，在对象销毁时自动调用

构造函数语法：
类名(){};
示例1：

	std::cout << "构造函数调用" << std::endl;
}
1.构造函数没有返回值也不写void;
2.构造函数名必须与类名相同，区分大小写；
3.构造函数可以有参数，也可以没有参数；可以重载
4.程序在调用对象时会自动调用构造函数，无需手动调用并且只调用一次；方法和常规函数一样

析构函数语法：
~类名(){};
1.析构函数没有返回值也不写void;
2.析构函数名必须与类名相同，前面加上波浪号~，区分大小写；
3.析构函数不能有参数，也不能重载；
4.程序在对象销毁时会自动调用析构函数，无需手动调用并且只调用一次；

## 什么是静态对象？

静态对象就是用 `static` 关键字修饰的对象。它的最大特点是：**生命周期贯穿整个程序运行期间**，从它被创建（通常是在第一次遇到它时）一直到程序结束才会被销毁。

你可以把静态对象想象成一个“长期居民”，而普通的局部对象（在函数内定义的对象）就像“临时访客”——函数调用结束它们就离开了。

## 静态对象的三种常见形式

### 1. 函数内的静态局部变量
```cpp
void counter() {
    static int count = 0;  // 静态局部变量
    count++;
    cout << "调用次数: " << count << endl;
}

int main() {
    counter(); // 输出 1
    counter(); // 输出 2
    counter(); // 输出 3
    return 0;
}
```
- 即使函数执行结束，`count` 也不会被销毁，下次调用时它保留着上次的值。
- 初始化只发生一次（第一次执行到该行时）。

### 2. 全局/命名空间作用域的静态对象（不推荐在头文件中使用）
```cpp
// File1.cpp
static int globalValue = 100;  // 只能在此文件中访问
```
- 它的作用域限制在当前编译单元（即当前 `.cpp` 文件），其他文件无法访问。
- 生命周期从程序启动到结束（但在 main 函数之前就构造好了）。

### 3. 类的静态成员变量
```cpp
class MyClass {
public:
    static int sharedValue;  // 类的静态成员变量，声明
};

// 在类外单独定义和初始化
int MyClass::sharedValue = 0;
```
- 这个变量属于类本身，而不是某个对象。所有对象共享同一个静态成员。
- 生命周期也是整个程序运行期间。

## 为什么要用静态对象？

1. **跨函数调用保持状态**  
   比如计数器、缓存等，希望数据在多次调用之间持续存在。

2. **实现单例模式**  
   让某个类只有一个实例，且全局可访问。

3. **共享数据**  
   类的静态成员可以让所有对象共享一份数据，减少内存开销。

4. **控制访问范围**  
   文件作用域的静态对象可以实现“文件内私有”的效果（类似于匿名命名空间）。

## 注意事项（简单了解）

- **初始化时机**：函数内的静态局部变量在第一次执行到它的定义时才初始化；全局/静态成员在 `main` 开始前就初始化（有复杂的初始化顺序问题，跨文件时需小心）。
- **线程安全**：C++11 保证了函数内静态局部变量的初始化是线程安全的，但手动管理的静态对象需要自己加锁。
- **不要过度使用**：静态对象会一直占用内存，而且可能使代码更难测试和理解。

## 总结

静态对象就是**生命周期贯穿程序始终的对象**，常用于需要跨函数/跨对象共享数据或保持状态的场景。你可以把它当作一个“永久存储”的小盒子，用的时候记得考虑初始化和线程安全问题。

希望这个解释对你有帮助！}

4.3 C++对象和this指针

4.3.1 成员变量和成员函数分别存储
在C++中类内成员变量和成员函数分别存储
只有在非静态成员变量才属于类的对象上
空对象占用为1字节，因为编译器会给每个对象都分配1字节空间，是为了区分空对象占用内存空间位置。
当类中存在只存在一个非静态int对象它只占用4字节因为它属于这个对象上
添加静态变量int时，只占用4字节因为静态成员变量不属于某个对象上，当继续添加非静态成员函数和静态成员函数，还是4字节因为它们不属于某个对象上

4.3.2 this指针概述
通过4.3.1我们知道在C++成员变量和成员函数是分开存储的
每一个非静态成员函数只会诞生一份函数实例，也就是多个同类型对象会共用一块代码
问题来了，那么这一块代码是怎么知道是哪个对象发起的调用呢？
C++通过提供特殊的对象指针，this指针，解决上述问题，this指针指向被调用的成员函数所属的对象

this指针是隐含每一个非静态成员函数的一种指针
this指针不需要定义，直接使用即可
this指针本质是一个指针常量，指向当前对象 指向不可修改(也就是指向的的对象不可改变)但是this指针的值可以修改(对象的成员属性可以修改例如this->age = 18)
相当于：类名 * const this;
如果想值也不被修改就像这样：const 类名 * const this;(对象和值都不可修改)
this指针的用途：
当形参和成员变量同名时，可以通过this指针区分
在类的非静态成员函数中返回对象本身，可以使用retuen *this;
示例：
class pero {
public:
	void setAge(int age) {
		this->age = age; // 通过this指针区分成员变量和形参
	}
	int getAge() {
		return this->age; // 通过this指针访问成员变量
	}
	pero& setHeight(int height) {
		this->height = height;
		return *this; // 返回对象本身
	}
private:
	int age;
	int height;
};

返回对象本身这种思维称为链式编程，链式编程就是通过返回对象本身，可以连续，调用对象的多个成员函数，主要为连续调用
使用值返回的方式会创建一个新对象，浪费内存空间，一般使用引用，返回对象本身
示例：
pero p;
p.setAge(18).setHeight(180); // 链式编程

有时候需要让某个函数或者类访问另一个类的私有成员，这时就需要用到友元
友元可以是函数，也可以是类
友元函数：通过在类中声明一个函数为友元函数，可以让该函数访问类的私有成员
友元类：通过在类中声明另一个类为友元类，可以让该类的所有成员函数访问本类的私有成员

生活中你的家有客厅，卧室，厨房等房间，每个房间都有自己的隐私
有时候你需要让你的好朋友进入你的卧室，这时你就需要给他一把钥匙
友元函数和友元类就像是钥匙，可以让外部函数或者类访问类的私有成员
在程序中，有一些私有属性也想让类外的特殊的函数或者类进行访问就需要用到友元技术
友元的目的就是为了让一个函数或类访问另一个类的私有成员

友元的关键字是friend
友元函数语法：在类中声明friend 返回值类型 函数名(参数列表);
友元类语法：在类中声明friend class 类名;

友元的三种实现：
全局函数做友元
类的成员函数做友元
类做友元

4.4.1 全局函数做友元
通过在类中声明一个全局函数为友元函数，可以让该函数访问类的私有成员
语法：在类中声明friend 返回值类型 函数名(参数列表); 相当于声明该函数可以访问类的私有成员
示例：
#include <iostream>
class Person {
private:
	std::string name;
	int age;
public:
	Person(std::string n, int a) :name(n), age(a) {}
	friend void showPerson(Person p); // 声明友元函数
};
void showPerson(Person p) { // 友元函数定义
	std::cout << "姓名：" << p.name << ", 年龄：" << p.age << std::endl; // 访问私有成员
}
int main() {
	Person p("张三", 18);
	showPerson(p); // 调用友元函数
	return 0;
}

4.4.2 类做友元
通过在类中声明另一个类为友元类，可以让该类的所有成员函数访问本类的私有成员
语法：在类中声明friend class 类名; 相当于声明该类的所有成员函数可以访问本类的私有成员
示例：
#include <iostream>
class Friend; // 前置声明,避免编译错误
class Person {
private:
	std::string name;
	int age;
public:
	Person(std::string n, int a) :name(n), age(a) {}
	friend class Friend; // 声明友元类
};
class Friend {
public:
	void showPerson(Person p) { // 友元类的成员函数
		std::cout << "姓名：" << p.name << ", 年龄：" << p.age << std::endl; // 访问私有成员
	}
};

类外写成员函数的方法：
例如现在有一个类A，我们想在类外实现它的成员函数func，可以按照以下步骤进行：
1. 在类A的声明中，先进行前置声明，告诉编译器类A的存在。
2. 在类A的声明中，声明成员函数func的原型。
3. 在类外实现成员函数func，使用类名加作用域运算符::来定义函数体。
示例代码如下：
#include <iostream>
class A; // 前置声明
class A {
public:
	void func(); // 成员函数声明
};
void A::func() { // 成员函数定义
	std::cout << "类A的成员函数func被调用" << std::endl;
}

4.4.3 成员函数做友元
通过在类中声明另一个类的成员函数为友元函数，可以让该成员函数访问本类的私有成员
语法：在类中声明friend 返回值类型 类名::函数名(参数列表);
示例：
#include <iostream>
class Person; // 前置声明
class Friend {
public:
	void showPerson(Person p); // 成员函数声明
};
class Person {
private:
	std::string name;
	int age;
public:
	Person(std::string n, int a) :name(n), age(a) {}
	friend void Friend::showPerson(Person p); // 声明成员函数为友元
};
void Friend::showPerson(Person p) { // 友元成员函数定义
	std::cout << "姓名：" << p.name << ", 年龄：" << p.age << std::endl; // 访问私有成员
}
int main() {
	Person p("张三", 18);
	Friend f;
	f.showPerson(p); // 调用友元成员函数
	return 0;
}

4.5 运算符重载
运算符重载概述
作用：让自定义类型的对象也能使用运算符进行操作，对已有的运算符进行重新定义和实现，赋予其新的功能，以应对不同的数据类型
语法：返回值类型 operator 运算符(参数列表) { 函数体; }
运算符重载的注意事项：
1.运算符重载不能改变运算符的优先级和结合性
2.运算符重载不能创建新的运算符
3.运算符重载必须至少有一个参数是自定义类型
4.运算符重载可以是成员函数，也可以是全局函数
5.运算符重载不能重载所有运算符，例如：sizeof、.:、.*、::等运算符不能重载
6.运算符重载可以发生函数重载
可以重载运算符如下：
// 一元运算符（9个）
+ - * & ~ ! ++ -- (类型转换运算符)

// 二元运算符（19个）
+ - * / % & | ^ << >> 
== != < > <= >= && || , ->*

// 赋值运算符（7个）
= += -= *= /= %= &= |= ^= <<= >>= 

// 特殊运算符（3个）
[] () ->
new       // 可以重载，但有一些限制
delete    // 可以重载，但有一些限制
new[]     // 可以重载数组版本
delete[]  // 可以重载数组版本

不可以重载如下：
.         // 成员访问运算符（对象）
.*        // 成员指针访问运算符（对象）
::        // 作用域解析运算符
?:        // 条件运算符（三元运算符）
sizeof    // 大小查询运算符
alignof   // 对齐查询运算符（C++11）
typeid    // 类型标识运算符
static_cast, dynamic_cast, const_cast, reinterpret_cast // 类型转换运算符
#         // 预处理器字符串化（不是真正的C++运算符）
##        // 预处理器标记连接（不是真正的C++运算符）

4.5.1 加号运算符重载
作用：重载加号运算符，使其能够对自定义类型的对象进行加法操作
示例：
#include <iostream>
class Point {
public:
	Point() {};
	Point(int a, int b) {
		this->m_a = a;
		this->m_b = b;
	}
	//成员函数实现加号运算符重载
	Point operator+(Point &p) {
	Point temp;
	temp.m_a = this->m_a + p.m_a;
	temp.m_b = this->m_b + p.m_b;
	return temp;
}
private:
	int m_a;
	int m_b;
};
int main() {
	Point p1(1, 2);
	Point p2(3, 4);
	Point p3 = p1 + p2; // 调用加号运算符重载
	return 0;
}
详细解释如下：
//成员函数实现加号运算符重载
Point operator+(Point &p) {
	Point temp;
	temp.m_a = this->m_a + p.m_a;
	temp.m_b = this->m_b + p.m_b;
	return temp;
}
相当于：
Point 自定义函数名(Point &p) {
	Point temp;
	temp.m_a = this->m_a + p.m_a;
	temp.m_b = this->m_b + p.m_b;
	return temp;
}
然后：
Point p3 = p1.自定义函数名(p2); // 这是未简化版的，本质
简化版只需要 Point p3 = p1 + p2; // 调用加号运算符重载
通过全局函数重载加号运算符
Point operator+(Point &p1, Point &p2) {
	Point temp;
	temp.m_a = p1.m_a + p2.m_a;
	temp.m_b = p1.m_b + p2.m_b;
	return temp;
}
本质：point p3 = operator+(p1,p2);
简化为：Point p3 = p1 + p2; // 调用加号运算符重载

4.5.2 左移运算符重载
作用：重载左移运算符，使其能够对自定义类型的对象进行输出操作

重载左移运算符其实就是：
告诉编译器"当有人想用<<输出我的对象时，应该怎么显示这个对象的内容"
就像给对象设计了一个"显示说明书"，这样cout就知道该怎么显示它了。

简单记忆法
记住这三句话：
<< 在输出时代表"流向"（数据流向输出设备）
左边是目的地（cout、文件流等）
右边是要输出的内容
一定要注意参数位置，因为左移运算符的左边是目的地，右边是内容，左边为输出流例如cout，右边为自定义类型对象
正确应该是：
ostream& operator<<(ostream &cout, Point &p) {
	// 实现输出逻辑
	return cout;
}
ostream& 为返回值类型，表示返回输出流对象的引用
ostream &cout 为第一个参数，表示输出流对象的引用
Point &p 为第二个参数，表示要输出的自定义类型对象的引用
return cout; 表示返回输出流对象的引用，方便链式调用,相当于将cout返回给下一个<<操作符使用
其中cout是引用时的别名，不一定非得是cout，可以是其他名字，但是习惯上一般用cout表示输出流对象

std::ostream& → 返回类型，必须是ostream的引用
operator<< → 函数名，表示重载<<运算符
(std::ostream &cout, Point &p) → 参数列表
第一个参数：输出流对象（这里起名叫cout，但可以是任意名字）
第二个参数：要输出的Point对象

为什么必须是全局函数？
因为左操作符不是你的类对象！
对于 cout << p，左操作数是ostream，右操作数是Point
如果是成员函数，应该这样调用：p.operator<<(cout)，不符合习惯
全局函数才能保持自然的语法：cout << p

通俗比喻
可以把 << 想象成一个传送带：
cout 是终点（屏幕）
数据通过传送带（<<）送到终点
每个 << 都在传送带上放一个数据包

示例：
#include <iostream>
class Point {
public:
	Point() {};
	Point(int a, int b) {
		this->m_a = a;
		this->m_b = b;
	}
	//友元函数声明左移运算符重载
	friend std::ostream& operator<<(std::ostream &cout, Point &p);
private:
	int m_a;
	int m_b;
};
//全局函数实现左移运算符重载
std::ostream& operator<<(std::ostream &os, Point &p) {
	os << "m_a=" << p.m_a << " m_b=" << p.m_b;
	return os;
}
int main() {
	Point p1(1, 2);
	std::cout << p1 << std::endl; // 调用左移运算符重载
	return 0;
}

4.5.3 重载递增运算符
作用：重载递增运算符，使其能够对自定义类型的对象进行递增操作
递增运算符有两种形式：
前置递增：++obj
后置递增：obj++
重载前置递增运算符
语法：返回值类型 operator++() { 函数体; }
示例：
#include <iostream>
class Counter {
public:
	Counter(int value) :m_value(value) {}
	// 重载前置递增运算符
	Counter& operator++() {
		++m_value;
		return *this; // 返回当前对象的引用
	}
	int getValue() const {
		return m_value;
	}
private:
	int m_value;
};
int main() {
	Counter c(10);
	++c; // 调用前置递增运算符重载
	std::cout << "前置递增后值为：" << c.getValue() << std::endl;
	return 0;
}
重载后置递增运算符
语法：返回值类型 operator++(int) { 函数体; }
示例：
#include <iostream>
class Counter {
public:
	Counter(int value) :m_value(value) {}
	// 重载后置递增运算符
	Counter operator++(int) {
		Counter temp = *this; // 保存当前状态
		++m_value; // 递增
		return temp; // 返回递增前的状态
	}
	int getValue() const {
		return m_value;
	}
private:
	int m_value;
};
int main() {
	Counter c(10);
	Counter oldC = c++; // 调用后置递增运算符重载
	std::cout << "后置递增前值为：" << oldC.getValue() << std::endl;
	std::cout << "后置递增后值为：" << c.getValue() << std::endl;
	return 0;
}
注意：
重载后置递增运算符时，必须添加一个int类型的参数，用于区分前置和后置递增，该参数不需要使用属于占位参数，原理是编译器通过参数个数来区分前置和后置递增，这就是函数重载
后置递增运算符通常返回递增前的对象状态，因此返回值类型一般为对象类型，而不是引用类型，因为返回的的是一个临时对象它会在函数结束后被销毁，而前置递增运算符通常返回当前对象的引用，以支持连续递增操作
当你疑惑为什么返回值类型是引用时，原因如下：
因为前置递增操作通常用于连续递增操作，例如 ++(++obj);
如果返回值不是引用，那么每次递增都会创建一个新的对象，导致源数据无法被修改,也就是无法实现连续递增
而返回引用则可以直接修改原对象，避免不必要的对象拷贝，提高效率
例如:
Counter& operator++() {
		++m_value;
		return *this; // 返回当前对象的引用
	}
这样可以支持连续递增操作，而不会创建多个临时对象

4.5.4 重载赋值运算符
作用：重载赋值运算符，使其能够对自定义类型的对象进行赋值操作
语法：返回值类型 operator=(const 类名 &对象) { 函数体; }
C++编译器至少为一个类添加四个函数
1.默认构造函数（无参，函数体为空）
2.默认析构函数（无参，函数体为空）
3.默认拷贝函数，对属性进行值拷贝 //值拷贝的意思为，将被拷贝的对象所以属性赋值给当前对象
4.默认赋值运算符重载，对属性进行值拷贝 //值拷贝的意思为，将被赋值的对象所以属性赋值给当前对象
如果类中有属性指向堆区内存，做赋值操作时就会出现深浅拷贝问题
因此需要自己提供赋值运算符重载函数，防止浅拷贝带来的问题
示例代码如下：
#include <iostream>
class Person {
public:
	Person(int age, int height, int weight) {
		this->age = age;
		this->height = height;
		this->weight = weight;
	}
	// 重载赋值运算符
	Person& operator=(const Person &p) {
		if (this != &p) { // 防止自我赋值
			this->age = p.age;
			this->height = p.height;
			this->weight = p.weight;
		}
		return *this; // 返回当前对象的引用
	}
	void setHeight(int h) { // 写权限
		if (h < 0 || h > 300) {
			std::cout << "身高数据不合法！" << std::endl;
			return;
		}
		height = h;
	}
	int getHeight() const { // 读权限
		return height;
	}
private:
	int age;
	int height;
	int weight;
};
int main() {
	Person p1(18, 180, 75);
	Person p2(20, 175, 70);
	p2 = p1; // 调用赋值运算符重载
	std::cout << "p2的身高为：" << p2.getHeight() << std::endl;
	return 0;
}

4.5.5 关系运算符重载
作用：重载关系运算符，可以让两个自定义类型对象进行对比操作
什么是关系运算符重载？
关系运算符重载就是让你自己创建的类型（比如类）也能使用 >, <, ==, !=, >=, <= 这些比较运算符，就像使用 int, double 这样的内置类型一样。

为什么需要？
当你创建了自己的数据类型（如学生类、日期类、分数类），想要比较两个对象时，如果不重载：

需要使用专门的函数：compare(a, b) 或 a.compareTo(b)
代码不够直观，使用起来麻烦
重载后可以直接：if (student1 > student2)，代码更简洁自然。
// ... existing code ...

#include <iostream>
using namespace std;

class Date {
private:
    int year;
    int month;
    int day;
    
public:
    // 构造函数
    Date(int y, int m, int d) : year(y), month(m), day(d) {}
    
    // ---------------------------
    // 1. 重载等于运算符 ==
    // ---------------------------
    bool operator==(const Date& other) const {
        return year == other.year && month == other.month && day == other.day;
    }
    
    // ---------------------------
    // 2. 重载不等于运算符 !=
    // 注意：通常用 == 来实现 !=
    // ---------------------------
    bool operator!=(const Date& other) const {
        return !(*this == other);  // 调用上面定义的 ==
    }
    
    // ---------------------------
    // 3. 重载小于运算符 <
    // ---------------------------
    bool operator<(const Date& other) const {
        if (year != other.year) return year < other.year;
        if (month != other.month) return month < other.month;
        return day < other.day;
    }
    
    // ---------------------------
    // 4. 重载小于等于运算符 <=
    // 注意：可以用 < 和 == 来实现
    // ---------------------------
    bool operator<=(const Date& other) const {
        return (*this < other) || (*this == other);
    }
    
    // ---------------------------
    // 5. 重载大于运算符 >
    // 注意：可以用 < 和 == 来实现
    // ---------------------------
    bool operator>(const Date& other) const {
        return !(*this <= other);
    }
    
    // ---------------------------
    // 6. 重载大于等于运算符 >=
    // 注意：可以用 < 来实现
    // ---------------------------
    bool operator>=(const Date& other) const {
        return !(*this < other);
    }
    
    // 显示日期
    void display() const {
        cout << year << "-" << month << "-" << day;
    }
};

int main() {
    Date today(2024, 3, 15);
    Date tomorrow(2024, 3, 16);
    Date sameDay(2024, 3, 15);
    
    // 使用重载的运算符进行比较
    cout << "today < tomorrow: " << (today < tomorrow) << endl;  // true (1)
    cout << "today > tomorrow: " << (today > tomorrow) << endl;  // false (0)
    cout << "today == sameDay: " << (today == sameDay) << endl;  // true (1)
    cout << "today != tomorrow: " << (today != tomorrow) << endl;// true (1)
    
    return 0;
}
4.5.6 重载函数调用运算符
函数调用运算符()也可以发生重载
由于重载后的使用方式非常像函数的调用所以称为仿函数
仿函数没有固定写法，非常灵活

作用：重载函数调用运算符，使得类的对象可以像函数一样被调用
语法：返回值类型 operator()(参数列表) { 函数体; }
示例：
#include <iostream>
class Adder {
public:
	Adder(int value) :m_value(value) {}
	// 重载函数调用运算符
	int operator()(int x) {
		return m_value + x;
	}
private:
	int m_value;	
};
int main() {
	Adder add5(5); // 创建一个Adder对象，初始值为5
	int result = add5(10); // 调用函数调用运算符重载，相当于 add5.operator()(10);
	std::cout << "结果为：" << result << std::endl; // 输出结果为15
	return 0;
}

在C++中，重载函数调用运算符（operator()）可以让类的对象像函数一样被调用，这类对象被称为函数对象或仿函数。

核心概念
当你重载了 operator() 运算符后，该类的对象就可以像函数一样使用 () 进行调用。

基本示例：
#include <iostream>
using namespace std;

class Adder {
private:
    int value;  // 可以保存状态
public:
    // 构造函数
    Adder(int val) : value(val) {}

    // 重载函数调用运算符
    int operator()(int x) {
        return value + x;
    }
};

int main() {
    Adder add5(5);  // 创建一个函数对象，初始值为5
    Adder add10(10); // 创建另一个函数对象，初始值为10
    
    cout << add5(3) << endl;  // 输出：8 (5 + 3)
    cout << add10(3) << endl; // 输出：13 (10 + 3)
    
    // 可以像普通函数一样使用
    int result = add5(7);  // result = 12
    
    return 0;
}
为什么使用函数对象？
1. 可以保存状态
普通函数没有状态，而函数对象可以有成员变量来保存状态：
class Counter {
private:
    int count;
public:
    Counter() : count(0) {}
    
    int operator()() {
        return ++count;
    }
};

int main() {
    Counter countFunc;
    cout << countFunc() << endl;  // 输出：1
    cout << countFunc() << endl;  // 输出：2
    cout << countFunc() << endl;  // 输出：3
    return 0;
}
2. 可以作为参数传递
经常与STL算法一起使用：
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

class IsGreaterThan {
private:
    int threshold;
public:
    IsGreaterThan(int t) : threshold(t) {}
    
    bool operator()(int x) {
        return x > threshold;
    }
};

int main() {
    vector<int> nums = {1, 5, 10, 15, 20};
    IsGreaterThan gt10(10);
    
    // 使用函数对象作为谓词
    int count = 0;
    for (int n : nums) {
        if (gt10(n)) {
            count++;
        }
    }
    cout << "大于10的元素个数：" << count << endl;
    
    // 或者使用STL算法
    auto it = find_if(nums.begin(), nums.end(), IsGreaterThan(15));
    if (it != nums.end()) {
        cout << "第一个大于15的元素：" << *it << endl;
    }
    
    return 0;
}
3. 可以有多个重载版本
可以重载不同参数列表的 operator()：
class MultiFunc {
public:
    int operator()(int a, int b) {
        return a + b;
    }
    
    double operator()(double a, double b) {
        return a * b;
    }
    
    void operator()(const string& message) {
        cout << "消息：" << message << endl;
    }
};

int main() {
    MultiFunc func;
    cout << func(5, 3) << endl;        // 输出：8
    cout << func(2.5, 4.0) << endl;    // 输出：10.0
    func("你好，世界！");           // 输出：消息：你好，世界！
    return 0;
}
与Lambda表达式的关系
C++11引入的lambda表达式本质上是匿名的函数对象：
int main() {
    // Lambda表达式
    auto lambda = [value = 5](int x) { return value + x; };
    cout << lambda(3) << endl;  // 输出：8
    
    // 等效的函数对象
    class LambdaEquivalent {
        int value;
    public:
        LambdaEquivalent(int val) : value(val) {}
        int operator()(int x) const { return value + x; }
    };
    
    LambdaEquivalent func(5);
    cout << func(3) << endl;  // 输出：8
    
    return 0;
}
超简单的理解方式：让对象“变身”成函数！
重载函数调用运算符就是给你的类对象增加一个“变身”能力 —— 让它能像普通函数一样被调用！

普通函数 vs 重载()的对象
正常函数是这样的：
int addFive(int x) {
    return 5 + x;
}
int main() {
    int result = addFive(10);  // 调用函数
    // result = 15
}
通过重载operator()，对象就能变成这样：
#include <iostream>
using namespace std;
// Step 1：创建一个"能变成函数的类"
class MagicAdder {
private:
    int baseValue;  // 记住的数字，比如5    
public:
    // 构造函数：告诉它要记哪个数字
    MagicAdder(int base) : baseValue(base) {}
    
    // Step 2：重载圆括号()！这是让对象变函数的关键！
    int operator()(int x) {
        return baseValue + x;  // 实现加法的逻辑
    }
};

int main() {
    // Step 3：创建对象（它现在是普通对象）
    MagicAdder add5(5);  // 这个对象记住了数字5
    
    // Step 4：魔法时刻！对象变身函数！
    // 看这里！add5 对象加上了圆括号 ()，像个函数一样被调用！
    int result = add5(10);  // 实际是 add5.operator()(10)
    // result = 15 (5 + 10)
    
    cout << "结果：" << result << endl;
    
    // 可以创建更多不同的"加法器"
    MagicAdder add100(100);
    cout << add100(3) << endl;    // 输出 103
    cout << add100(20) << endl;   // 输出 120
    
    return 0;
}

4.6 继承
继承是面向对象的三大特性之一
例如：动物是一个类，狗和猫都是动物的子类，狗和猫都继承了动物的属性和行为
继承的作用：代码重用，子类可以直接使用父类的属性和方法，避免重复代码
子类也被称为派生类，父类也被称为基类
继承的语法：class 子类名 : 访问权限 父类名 {
	// 子类的成员
};
访问权限：public、protected、private
访问权限的作用：控制子类对父类成员的访问权限
访问权限的区别：
public：子类可以访问父类的public成员，无法访问父类的protected和private成员但是可以将父类的private成员使用public的封装方法进行访问示例（继承public）
protected：子类可以访问父类的public和protected成员
private：子类只能访问父类的public成员，无法访问父类的protected和private成员
继承访问级别的对比表格
访问限定符	类内部	  子类	     类外部
private	  ✓ 可访问	✗ 不可访问	✗ 不可访问
protected ✓ 可访问  ✓ 可访问	✗ 不可访问
public	  ✓ 可访问	✓ 可访问	✓ 可访问
总结要点
私有成员是严格封装的：子类不能直接访问父类的私有成员
通过公有接口访问：子类通过调用父类的公有成员函数（如 getid()、getname()）来间接访问私有数据
设计原则：这是面向对象设计中的封装原则，保护数据不被意外修改
如果需要直接访问：可以考虑使用 protected 访问级别，但这会降低封装性
**结论先行**：子类确实**继承了基类的所有成员（包括私有成员）**，但**不能直接访问**基类的私有成员。
---
### 理解“继承”和“访问权限”

1. **继承的本质**  
   当子类继承父类时，子类对象内部会包含一个完整的父类子对象。也就是说，父类的所有数据成员（无论是什么访问权限）都实实在在地存在于子类对象的内存中。例如：

   ```cpp
   class Employee {
   private:
       string name;
       int id;
   };

   class Manager : public Employee {
       double bonus;
   };
   ```

   创建一个 `Manager` 对象时，内存中会包含 `name`、`id` 和 `bonus` 三个成员变量。所以从“包含关系”上说，子类确实继承了父类的所有成员。

2. **访问权限控制的是“能不能直接使用”**  
   - `private` 成员：只能在**本类**的成员函数中直接访问，派生类（子类）**不能直接访问**。  
   - `protected` 成员：可以在本类和派生类中直接访问，但不能在类外访问。  
   - `public` 成员：任何地方都可以直接访问。

   因此，虽然 `Manager` 对象内部有 `name` 和 `id`，但在 `Manager` 的成员函数中不能直接写 `name = "李四"` 或 `cout << id`，因为它们是 `Employee` 的私有成员。必须通过 `Employee` 提供的公有接口（如 `getName()`）来间接访问。

3. **为什么这样设计？**  
   这是**封装**的体现：父类不想让子类随意修改自己的私有数据，强制子类通过父类提供的公有方法来操作数据，从而保证父类的内部状态不会被破坏。

---
### 用代码验证

```cpp
class Employee {
private:
    string name;   // 私有成员
    int id;
public:
    Employee(string n, int i) : name(n), id(i) {}
    string getName() const { return name; }
    int getId() const { return id; }
};

class Manager : public Employee {
public:
    Manager(string n, int i, double b) : Employee(n, i), bonus(b) {}
    void show() {
        // cout << name;     // 错误！不能直接访问私有成员 name
        // cout << id;       // 错误！不能直接访问私有成员 id
        cout << getName();    // 正确，通过公有成员函数访问
        cout << getId();      // 正确
        cout << bonus;        // 正确，bonus 是自己的成员
    }
private:
    double bonus;
};
```
---
### 总结
- 继承是“**拥有**”关系：子类拥有父类的所有成员（包括私有）。  
- 访问权限是“**可见**”规则：私有成员对子类不可见（不能直接使用），但可以通过父类提供的公有/保护接口间接使用。  
- 所以，子类确实继承了父类的私有成员，只是不能直接碰它们。

示例（继承public）:

4.6.2 继承方式
继承的语法：class 子类 : 继承方式 父类
继承方式一共有三种：
public继承：父类的public成员在子类中仍然是public，父类的protected成员在子类中仍然是protected，父类的private成员在子类中仍然是private
protected继承：父类的public成员在子类中变为protected，父类的protected成员在子类中仍然是protected，父类的private成员在子类中仍然是private
private继承：父类的public成员在子类中变为private，父类的protected成员在子类中变为private，父类的private成员在子类中仍然是private
继承方式的选择：
如果希望子类能够直接访问父类的public成员，应该选择public继承
如果希望子类能够访问父类的public和protected成员，但不希望外部访问父类的public成员，应该选择protected继承
如果希望子类不能访问父类的public和protected成员，应该选择private继承

4.6.3 继承中的对象模型
在C++中，继承关系会影响对象的内存布局。
子类对象会包含一个父类子对象，这个父类子对象占用内存空间来存储父类的成员变量。
无论访问权限如何，父类的成员变量都会存在于子类对象的内存中。
父类中所有的非静态成员变量都会被子类对象继承下来，并占用内存空间。静态成员变量不属于对象实例，而是属于类，因此不会被继承到子类对象中。

使用vs查看对象内存布局操作：
打开开始菜单，找到你安装的vs版本，打开vs开发者命令提示符，跳转到要查看的文件的盘符,例如D盘，输入D:回车，
然后输入cd 路径回车，例如cd D:\cpp\project\，
最后输入cl /d1 reportSingleClassLayout<类名> 文件名.cpp回车，就可以看到对象内存布局了

4.6.4 继承中的构造函数和析构函数
在继承关系中，构造函数和析构函数的调用顺序是固定的：
构造函数调用顺序：先调用父类的构造函数，再调用子类的构造函数
析构函数调用顺序：先调用子类的析构函数，再调用父类的析构函数
构造函数的调用方式：
子类的构造函数可以通过成员初始化列表的方式调用父类的构造函数，例如：子类构造函数(参数列表) : 父类构造函数(参数列表) { 函数体; }
如果子类构造函数没有显式调用父类的构造函数，编译器会自动调用父类的默认构造函数（无参构造函数）。如果父类没有默认构造函数，必须在子类构造函数中显式调用父类的构造函数，否则编译会报错。
析构函数的调用方式：
析构函数不需要显式调用，编译器会自动按照调用顺序调用析构函数，确保资源的正确释放。

4.6.5 继承同名成员的处理
同名成员函数：当父类和子类中存在同名成员函数时，子类会隐藏父类的所有同名成员函数包括重载后的，这种现象称为名字隐藏。
访问本类的同名成员函数：直接使用成员函数名即可访问子类的同名成员函数，例如：成员函数名()
访问父类的同名成员函数：
可以通过作用域解析运算符来访问父类的同名成员函数，例如：父类名::成员函数名()
当父类和子类中存在同名成员时，子类会隐藏父类的同名成员，这种现象称为名字隐藏。
访问本类的同名成员：直接使用成员名即可访问子类的同名成员，例如：成员名
访问父类的同名成员：
可以通过作用域解析运算符来访问父类的同名成员，例如：父类名::成员名
示例：
#include <iostream>
class Base {
public:
	int value;
	Base(int v) : value(v) {}
};
class Derived : public Base {
public:
	int value; // 同名成员
	Derived(int baseValue, int derivedValue) : Base(baseValue), value(derivedValue) {}
	void display() {
		std::cout << "Base value: " << Base::value << std::endl; // 访问父类的同名成员
		std::cout << "Derived value: " << value << std::endl;   // 访问子类的同名成员
	}
};
int main() {
	Derived d(10, 20);
	d.display();
	return 0;
}

4.6.6 继承同名静态成员的处理
同名静态成员：当父类和子类中存在同名静态成员时，子类会隐藏父类的同名静态成员，这种现象称为名字隐藏。
访问本类的同名静态成员：直接使用成员名即可访问子类的同名静态成员，例如：成员名
访问父类的同名静态成员：
可以通过作用域解析运算符来访问父类的同名静态成员，例如：父类名::成员名
如果想通过子类再去访问父类的同名静态成员，可以使用作用域解析运算符来指定访问父类的成员，例如：
子类名::父类名::成员名
解释如下：子类名::的意思为通过子类来访问成员，父类名::的意思是通过父类来访问成员，成员名为要访问的同名静态成员的名字
实施路径为：通过子类来访问父类的同名静态成员，首先通过子类名::来指定访问子类的成员，然后通过父类名::来指定访问父类的成员，最后是成员名来指定要访问的同名静态成员

在子类出现同名的静态成员函数时，子类的同名静态成员函数会隐藏父类所有的同名静态成员函数，这时如果想要访问父类的同名静态成员函数，可以通过作用域解析运算符来指定访问父类的成员，例如：
子类名::父类名::静态成员函数名

在 C++ 中，派生类可以继承基类的静态成员，但需要注意以下几点：
1. **静态成员属于类，而非对象**  
   静态成员（静态变量和静态函数）是在整个继承层次中共享的，所有派生类都访问同一个实例。
2. **派生类中可见性**  
   如果基类的静态成员是 `public` 或 `protected` 的，派生类可以直接通过类名或对象访问它们。例如：
   class Base {
   public:
       static int value;
   };
   int Base::value = 0;

   class Derived : public Base {};

   int main() {
       Derived::value = 42;           // 通过派生类名访问
       Derived d;
       d.value = 100;                  // 通过对象访问
       return 0;
   }
   ```
3. **没有独立副本**  
   派生类不会获得基类静态成员的独立副本，修改该静态成员会影响整个继承体系中的所有类。
4. **访问控制**  
   如果基类静态成员是 `private` 的，派生类无法直接访问，但可以通过基类的公有或受保护静态成员函数间接操作。
5. **隐藏与重定义**  
   如果派生类定义了同名的静态成员，则会隐藏基类的静态成员。此时可以通过作用域解析运算符 `Base::` 来访问基类的版本。
总结：派生类可以“继承”基类的静态成员，但这里的继承主要体现在访问权限和名称查找上，实际存储只有一份，被所有派生类共享。

4.6.7 多继承语法
c++中运行继承多个父类
语法：class 子类 : 继承方式 父类1, 继承方式 父类2, ... {// 子类的成员};
多继承容易导致多个父类存在同名成员的情况，这时候需要加作用域解析运算符来指定访问哪个父类的成员，
例如：父类名::成员名 
c++实际开发中不推荐使用多继承，因为多继承会增加代码的复杂度，容易引入二义性问题和维护困难的问题，建议使用接口类或者组合的方式来实现类似的功能。

4.6.8 菱形继承问题
菱形继承是指一个子类同时继承自两个父类，而这两个父类又继承自同一个祖先类，形成一个菱形的继承结构。
两个派生类继承同一个基类，又有某个类同时继承了这两个派生类，这就是菱形继承也称为钻石继承。
菱形继承会导致二义性问题，因为子类无法确定应该使用哪个父类的成员，编译器会报错。
解决菱形继承问题的方法是使用虚继承，虚继承可以确保在菱形继承结构中，只有一个基类的实例被创建，子类共享这个基类的实例，从而避免了二义性问题。
虚继承之后无论怎么修改都是同一份数据了，避免了二义性问题，节省了内存空间，同时也简化了代码的维护。
使用虚继承的时候，基类被声明为虚基类，这样在派生类中就会共享同一个基类的实例，避免了二义性问题。
虚继承的语法：class 子类 : 继承方式 virtual 父类 {
	// 子类的成员
};
在使用虚继承以后再查看底层时会发现，多了一个vbptr指针，这个指针是用来指向虚基类的实例的，确保在菱形继承结构中只有一个基类实例被创建和共享。
被称为虚指针（vbptr），它是一个指向虚基类实例的指针，用于解决菱形继承中的二义性问题，确保在菱形继承结构中只有一个基类实例被创建和共享。
vbptr指向了虚基类表（vbtbl），虚基类表中存储了虚基类的实例的地址，这样在访问虚基类的成员时就可以通过vbptr来找到正确的实例，避免了二义性问题。
当最后的类继承了两个派生类时，两个派生类继承了同一个基类并且两个派生类使用了虚继承，最后的类继承到的只是一个vbptr指针，指向了同一个基类的实例

在菱形继承中，当两个父类拥有同一份数据时需要加以作用域进行区分，例如：父类名::成员名
菱形继承会导致以下问题：
1. 二义性问题：当子类同时继承自两个父类，而这两个父类又继承自同一个祖先类时，子类无法确定应该使用哪个父类的成员，编译器会报错。
2. 内存浪费：如果不使用虚继承，子类会创建两个父类的实例，导致内存浪费。
3. 维护困难：菱形继承会增加代码的复杂度，容易引入错误，维护困难。

4.7 多态
4.7.1 多态的基本概念
多态是面向对象的三大特性之一
多态的作用：允许不同类型的对象通过相同的接口进行操作，提高代码的灵活性和可扩展性
多态的实现方式：通过虚函数实现 
多态的类型：编译时多态（函数重载和运算符重载）和运行时多态（派生类和虚函数），也就是静态多态和动态多态
编译时多态：函数重载和运算符重载是在编译阶段根据参数类型和数量来决定调用哪个函数的，这种多态称为译时静态多态
运行时多态：虚函数是在运行阶段根据对象的实际类型来决定调用哪个函数的，这种多态称为动态多态
静态多态：静态多态的函数地址早绑定，在编译阶段就确定了函数调用的地址，无法改变
动态多态：动态多态的函数地址晚绑定，在运行阶段才确定了函数调用的地址，可以根据对象的实际类型来调用不同的函数实现
多态的实现机制：虚函数表和虚指针
虚函数表（vtable）是一个指针数组，每个类都有一个虚函数表，存储了该类的虚函数的地址
虚指针（vptr）是一个指向虚函数表的指针，每个对象都有一个虚指针，指向所属类的虚函数表
当通过基类指针或引用调用虚函数时，编译器会通过虚指针找到对应的虚函数表，然后调用正确的函数实现，实现了运行时多态

在函数前加上关键字virtual就可以将函数声明为虚函数，虚函数是实现多态的关键
虚函数语法：virtual 返回值类型 函数名(参数列表) { 函数体; }
虚函数的特点：
1. 虚函数必须在基类中声明，派生类可以选择重写（覆盖）虚函数，也可以不重写，如果派生类不重写虚函数，则会继承基类的虚函数实现
2. 虚函数必须是成员函数，不能是非成员函数
3. 虚函数可以有默认参数，但在调用时必须使用基类指针或引用来调用虚函数，才能实现多态效果
4. 虚函数可以是纯虚函数，纯虚函数没有函数体，必须在派生类中重写纯虚函数，否则派生类也会成为抽象类，无法实例化对象
纯虚函数语法：virtual 返回值类型 函数名(参数列表) = 0;
抽象类是包含纯虚函数的类，抽象类无法实例化对象，只能作为基类被派生类继承，派生类必须重写抽象类中的纯虚函数才能实例化对象
抽象类的作用：抽象类可以作为接口类，定义一组接口供派生类实现，抽象类也可以提供一些默认的实现供派生类使用
抽象类的语法：class 类名 { public: virtual 返回值类型 函数名(参数列表) = 0; // 纯虚函数 };

动态多态满足条件：
1. 需要有继承关系：多态发生在基类和派生类之间，派生类必须继承自基类
2. 需要有虚函数：基类中必须至少有一个虚函数，派生类可以重写这个虚函数来提供不同的实现，子类必须重写父类虚函数才能实现多态效果，如果子类不重写父类虚函数，则会继承父类的虚函数实现
3. 需要通过基类指针或引用调用虚函数：只有通过基类指针或引用来调用虚函数时，才会发生动态多态，编译器会根据对象的实际类型来决定调用哪个函数实现
父类指针或引用 执行子类对象

重写：返回值类型 函数名 参数列表 完全相同才能算是重写，子类重写时virtual关键字可写可不写，建议写上virtual关键字以增强代码的可读性和维护性，
父类的虚函数必须是public或protected的，不能是private的，否则子类无法重写父类的虚函数
父类必须写上virtual关键字来声明虚函数，子类重写时可以选择写上virtual关键字，但不写也算是重写，建议写上virtual关键字以增强代码的可读性和维护性
重载：函数名相同，参数列表不同（参数类型、参数数量、参数顺序至少有一个不同）就算是重载
重写和重载的区别：
重写（Override）是指在派生类中重新定义一个与基类中虚函数完全相同的函数，以提供不同的实现。重写要求函数名、参数列表和返回值类型必须完全相同。
重载（Overload）是指在同一个作用域内定义多个函数，这些函数具有相同的函数名，但参数列表不同（参数类型、参数数量或参数顺序至少有一个不同）。重载不要求返回值类型相同。
重写和重载的关系：
重写和重载是两种不同的函数多态机制，重写发生在继承关系中，重载发生在同一作用域内。重写是为了实现运行时多态，而重载是为了提供编译时多态。

## 动态创建对象（简单易懂版）
### 什么是动态创建对象？
动态创建对象就是在程序**运行期间**（而不是编译时）根据需要临时创建对象。  
我们之前创建对象的方式大多是：
```cpp
Dog dog;          // 静态创建（栈上创建）
```
这种方式叫**静态创建**，对象在定义时自动分配内存，作用域结束后自动销毁。

而**动态创建**则是用 `new` 关键字在**堆内存**上手动创建对象，并且需要自己用 `delete` 手动释放内存。
---
### 为什么需要动态创建对象？
1. **对象数量不确定**：比如程序运行时才知道要创建几个动物，用静态数组无法灵活处理。
2. **对象生命周期需要手动控制**：对象可以在函数结束后仍然存在（比如返回给调用者）。
3. **多态的必要条件**：要实现**基类指针指向派生类对象**，通常必须动态创建（因为派生类对象的大小在编译时未知，但基类指针可以指向任何派生类，且对象必须存活到使用结束）。
### 动态创建的语法
#### 1. 创建单个对象
```cpp
// 语法：new 类型(初始值)
int* p = new int(10);          // 动态创建一个 int，初始值 10
dog* myDog = new dog();        // 动态创建一个 dog 对象，调用默认构造函数

- `new` 返回的是**指针**，指向新创建的对象。
- 如果类有构造函数，可以传参：`new dog("旺财")`。
#### 2. 创建对象数组

dog* dogs = new dog[3];        // 动态创建 3 个 dog 对象（调用默认构造函数 3 次）

注意：创建数组时不能直接给每个元素传参（除非使用 C++11 的列表初始化，但初学者先掌握基础）。
---
### 动态创建的对象怎么使用？
通过指针来操作：
dog* p = new dog();
p->speak();        // 用 -> 调用成员函数
(*p).speak();      // 等价写法（先解引用再用点）
p->age = 3;        // 访问成员变量（假设 public）
```
在多态中，我们通常用基类指针来操作派生类对象：
Animal* a = new dog();
a->speak();        // 调用的是 dog 的 speak()（多态）
---
### 必须手动释放内存！
动态创建的对象不会自动销毁，必须用 `delete` 手动释放，否则会造成**内存泄漏**（程序一直占用内存，越跑越慢）。
#### 释放单个对象
delete p;          // p 是之前 new 返回的指针
p = nullptr;       // 好习惯：释放后把指针置空，避免误用
#### 释放对象数组
delete[] dogs;     // 注意 [] 不能少！
---
### 结合多态的注意事项
当用基类指针指向派生类对象时，**释放时必须确保析构函数是虚函数**，否则只会调用基类的析构函数，导致派生类的资源可能没释放完。
class Animal {
public:
    virtual void speak() = 0;
    virtual ~Animal() {}   // 虚析构函数！
};
如果忘了写虚析构，通过 `Animal*` 删除派生类对象时，派生类的析构函数不会被调用，可能导致内存泄漏（如果派生类有自己的动态资源）。
刚学的阶段可以先记住这个规则，后面会深入理解。
---
### 静态创建 vs 动态创建 对比

| 特性               | 静态创建                         | 动态创建                     |
| ------------------ | -------------------------------- | ---------------------------- |
| 创建方式           | `Dog dog;`                       | `Dog* p = new Dog();`        |
| 内存位置           | 栈（stack）                       | 堆（heap）                   |
| 生命周期           | 作用域结束时自动销毁             | 手动用 `delete` 销毁         |
| 大小               | 编译时已知                       | 运行时分配                   |
| 多态支持           | 一般用指针或引用才能实现多态     | 天然适合，基类指针指向派生类 |
| 适用场景           | 简单、局部、数量固定             | 数量动态、需跨函数、多态     |

--

### 总结

- **动态创建对象** = 用 `new` 在堆上建对象，返回指针。
- **必须配对 `delete`**，否则内存泄漏。
- **多态必须配合指针/引用 + 虚函数**，动态创建是常用手段。
- **基类析构函数要写虚**，否则删除派生类对象可能出问题。

动态创建并不难，关键是要养成**“谁 new 谁 delete”**的好习惯。每次写 `new` 时，马上想好在哪里 `delete`。刚开始可以多写一些测试程序，感受内存的变化（比如用 `system("pause")` 观察内存占用）。

多态的深度剖析：
在类中有一个非静态函数也不是虚函数时，整个类属于是空类占1字节，当该非静态函数被声明为虚函数时，会发现该类占用变成了4字节
因为当一个类里有了虚函数，编译器会给每个对象添加一个隐藏的指针（通常叫 vptr），这个指针的大小（在你的环境下是 4字节）就决定了对象的大小。
这个隐藏的指针叫做虚指针（vptr），它指向一个叫做虚函数表（vtable）的结构，虚函数表中存储了该类的虚函数的地址。当通过基类指针调用虚函数时，编译器会通过 vptr 找到对应的 vtable，然后调用正确的函数实现，实现了运行时多态。
当一个类中有虚函数时，每个对象都会有一个 vptr 指针，这个指针占用内存空间，所以对象的大小会增加。即使该类没有其他成员变量，单单因为有了虚函数，编译器也会为每个对象添加一个 vptr 指针，使得对象的大小至少为指针的大小（在你的环境下是 4字节）。因此，当你将一个非虚函数声明为虚函数时，对象的大小就从 1字节增加到了 4字节。
虚指针和虚函数表指针是同一个东西，只不过是不同的称呼。
虚函数表内记录类似于：
[0] &Base::speak
[1] &Base::eat
像这样子记录着虚函数的地址，构成：&类名::函数名
当子类继承父类但未重写父类的虚函数时，子类对象的 vptr 仍然指向父类的 vtable，因此调用虚函数时会执行父类的实现。
当子类重写了父类的虚函数时，子类对象的 vptr 会指向子类自己的 vtable，子类的 vtable 中会记录子类重写后的虚函数地址，因此调用虚函数时会执行子类的实现。
当通过基类指针调用虚函数时，编译器会根据对象的实际类型来决定调用哪个函数实现，这就是运行时多态的核心机制。

多态的好处如下：
1. **接口统一便于维护**：通过基类指针或引用调用虚函数，可以使用相同的接口操作不同类型的对象，组织结构清晰，代码更简洁。
2. **可扩展性强**：添加新的派生类时无需修改现有代码，只需重写虚函数即可，符合开闭原则。
3. **实现动态绑定**：在运行时根据对象的实际类型调用正确的函数实现，实现了动态行为。
4. **提高代码的灵活性**：多态允许我们编写更通用的代码，可以处理更多类型的对象，增强了代码的灵活性和适应性。
5. **支持抽象类和接口**：通过纯虚函数和抽象类，可以定义接口规范，强制派生类实现特定的功能，增强了代码的规范性和可维护性。
6.可读性强：多态使得代码更易读，因为我们可以通过基类指针或引用来操作不同类型的对象，代码更简洁明了，容易理解。
在开发中提倡开闭原则，
开闭原则是指软件实体（类、模块、函数等）应该对扩展开放，对修改关闭。
也就是说，当需要添加新功能时，应该通过添加新的代码来实现，而不是修改现有的代码。
这样可以避免引入新的错误，保持代码的稳定性和可靠性。
多态是实现开闭原则的重要手段，通过使用虚函数和基类指针，我们可以在不修改现有代码的情况下添加新的派生类来扩展功能，从而实现开闭原则。

除了用父类的指针和引用调用子类的虚函数还可以使用父类指针直接赋值子类对象来调用子类的虚函数，例如：
Base* p = new Derived();

使用new创建类内对象语法：
new 类名; 或者 new 类名(参数列表);或者 new 类名();
new 类名()和new 类名;的区别：
new 类名()会调用类的默认构造函数来初始化对象，如果类没有定义默认构造函数，编译器会自动生成一个默认构造函数来初始化对象的成员变量为默认值（基本类型为0，指针为nullptr，类类型调用默认构造函数）
new 类名;不会调用类的构造函数来初始化对象，创建的对象的成员变量的值是未定义的，可能是随机值或者垃圾值，因此使用new 类名;创建对象时需要注意初始化对象的成员变量，否则可能会导致程序出现不可预知的行为。
new 类名(参数列表)会调用类的有参构造函数来初始化对象，如果类没有定义对应参数列表的构造函数，编译器会报错。

4.7.3 抽象类和虚函数
抽象类是包含纯虚函数的类，纯虚函数没有函数体，必须在派生类中重写纯虚函数，否则派生类也会成为抽象类，无法实例化对象
抽象类的作用：抽象类可以作为接口类，定义一组接口供派生类实现，抽象类也可以提供一些默认的实现供派生类使用
抽象类的语法：class 类名 { 访问权限: virtual 返回值类型 函数名(参数列表) = 0; // 纯虚函数 };
纯虚函数的特点：
1. 纯虚函数没有函数体，必须在派生类中重写纯虚函数，否则派生类也会成为抽象类，无法实例化对象
2. 纯虚函数可以有默认参数，但在调用时必须使用基类指针或引用来调用纯虚函数，才能实现多态效果
3. 纯虚函数可以在抽象类中提供默认的实现供派生类使用，派生类可以选择重写这个纯虚函数来提供不同的实现，也可以直接使用抽象类中提供的默认实现
抽象类的特点：不能实例化对象，通常用作基类来定义接口和提供默认实现
只要有一个纯虚函数的类就是抽象类，抽象类不能实例化对象，但可以定义指向抽象类的指针或引用，指向派生类对象来实现多态

4.7.5 虚析构与纯虚析构
在使用多态时，如果有子类将数据开辟到堆区就会导致父类指针无法调用到子类的虚构函数来释放子类的资源，造成内存泄漏的问题。
因此，在基类中将析构函数声明为虚函数为纯虚函数，让派生类必须重写析构函数来正确释放资源，
可以确保在通过基类指针删除派生类对象时，能够正确调用派生类的析构函数，确保资源的正确释放。

虚析构与纯虚析构的共性：
可以解决父类指针删除子类对象时无法调用子类析构函数的问题，确保资源的正确释放
都需要在基类中声明析构函数为虚函数或纯虚函数，派生类必须重写析构函数来正确释放资源
两个的区别：
如果是纯虚析构函数，基类就成为抽象类，无法实例化对象，但派生类必须重写纯虚析构函数才能实例化对象
如果是虚析构函数，基类可以实例化对象，派生类可以选择重写虚析构函数来提供不同的实现，也可以直接使用基类中提供的默认实现

虚析构函数是指在基类中将析构函数声明为虚函数，
这样在通过基类指针删除派生类对象时，能够正确调用派生类的析构函数，确保资源的正确释放。
虚析构函数的语法：virtual ~类名() { 函数体; }

纯虚析构函数是指在基类中将析构函数声明为纯虚函数，然后再类外单独提供一个函数体来实现析构操作，
这样基类就成为抽象类，无法实例化对象，但派生类必须重写纯虚析构函数才能实例化对象。
这样是因为如果基类有开辟堆区资源的操作必须要走析构函数来释放资源，
如果基类的析构函数是纯虚函数，那么在删除派生类对象时，
编译器会先调用派生类的析构函数来释放派生类的资源，然后再调用基类的析构函数来释放基类的资源，确保资源的正确释放。

纯虚析构函数的语法：
基类内：virtual ~类名() = 0; 
类外：类名::~类名() { 函数体; } // 纯虚析构函数必须有函数体
请记住这两个是分开的，不能合在一起写成：virtual ~类名() = 0 { 函数体; }，这样会导致编译错误，
因为纯虚析构函数必须先声明为纯虚函数，然后再提供一个单独的函数体来实现析构操作。
类似于静态成员函数的定义方式

纯虚析构函数必须有函数体，虽然它被声明为纯虚函数，但仍然需要提供一个函数体来实现析构操作。

纯虚析构函数的函数体可以是空的，也可以包含一些清理资源的代码，例如：
virtual ~类名() = 0 {
	// 清理资源的代码
}
纯虚析构函数的作用：
1. 使基类成为抽象类，无法实例化对象，只能作为基类被派生类继承
2. 强制派生类重写纯虚析构函数，确保派生类的资源能够正确释放
3. 在通过基类指针删除派生类对象时，能够正确调用派生类的析构函数，避免内存泄漏和资源未释放的问题
使用虚析构函数和纯虚析构函数的注意事项：
1. 如果基类中有虚函数，建议将析构函数也声明为虚函数，以确保通过基类指针删除派生类对象时能够正确调用析构函数。
2. 如果基类中有纯虚函数，建议将析构函数也声明为纯虚函数，以确保派生类必须重写析构函数来正确释放资源。
3. 在派生类中重写纯虚析构函数时，必须提供函数体来实现析构操作，不能留空，否则会导致编译错误。

5. 文件操作
程序在运行中产生的数据是临时的，程序结束后就会消失，如果想要保存数据，
就需要将数据写入到文件中，文件是存储数据的载体，可以在程序运行结束后仍然存在。通过文件可以做到数据的持久化存储。
在c++中，文件操作主要通过文件流来实现，文件流是c++提供的一套用于处理文件输入输出的类库，位于头文件<fstream>中。
文件流的分类：
输入文件流（ifstream）：用于从文件中读取数据
输出文件流（ofstream）：用于向文件中写入数据
读写文件流（fstream）：既可以从文件中读取数据，也可以向文件中写入数据
文件流的使用步骤：
1. 包含头文件：#include <fstream>
2. 创建文件流对象：根据需要创建输入文件流对象、输出文件流对象或读写文件流对象，
例如：std::ifstream inFile; std::ofstream outFile; std::fstream file;
3. 打开文件：使用文件流对象的open()成员函数打开文件，
例如：inFile.open("input.txt"); outFile.open("output.txt"); file.open("data.txt");
4. 进行文件操作：使用文件流对象的成员函数进行读写操作，
例如：inFile >> data; outFile << data; file << data;
5. 关闭文件：使用文件流对象的close()成员函数关闭文件，
例如：inFile.close(); outFile.close(); file.close();
文件流的成员函数：
open()：打开文件，需要传入文件名和打开模式（可选），例如：inFile.open("input.txt", std::ios::in);返回值类型为void，如果打开文件失败会设置文件流的状态标志，
可以通过is_open()成员函数来检查文件是否成功打开，
close()：关闭文件，例如：inFile.close();
is_open()：检查文件是否成功打开，返回值为true表示成功打开，否则返回false
good()：检查文件流是否处于良好状态，返回值为true表示文件流处于良好状态，否则返回false
eof()：检查是否到达文件末尾，返回值为true表示到达文件末尾，否则返回false
fail()：检查文件流是否发生错误，返回值为true表示文件流发生错误，否则返回false
bad()：检查文件流是否发生严重错误，返回值为true表示文件流发生严重错误，否则返回false
getline(test, content)：从输入流中读取一行文本，存储到content字符串中，返回值为输入流对象的引用，可以用于链式调用或条件判断
文件流的读写操作：
使用输入文件流对象的成员函数进行读取操作，例如：inFile >> data; inFile.getline(buffer, size);
使用输出文件流对象的成员函数进行写入操作，例如：outFile << data; outFile.write(buffer, size);
使用读写文件流对象的成员函数进行读写操作，例如：file >> data; file << data; file.getline(buffer, size); file.write(buffer, size);
文件流的异常处理：
在进行文件操作时，可能会遇到各种异常情况，例如文件不存在、权限不足、磁盘空间不足等，需要进行异常处理来保证程序的健壮性。
可以使用文件流对象的成员函数来检查文件流的状态，例如：if (!inFile.is_open()) { // 处理文件打开失败的情况 }
也可以使用异常处理机制来捕获文件操作中的异常，例如：
try {
	std::ifstream inFile("input.txt");
	if (!inFile.is_open()) {
		throw std::runtime_error("无法打开文件");
	}
	// 进行文件操作
	inFile.close();
} catch (const std::exception& e) {
	std::cerr << "发生异常: " << e.what() << std::endl;
}

打开模式（openmode）
ios_base::openmode 定义了以下常用标志（位于 std::ios 或 std::ios_base 命名空间中）：
标志	含义	适用流类型
std::ios::in	以读取方式打开文件。如果文件不存在，打开失败。	ifstream, fstream
std::ios::out	以写入方式打开文件。默认会清空文件原有内容（如果文件存在）。	ofstream, fstream
std::ios::app	以追加方式写入。所有输出都添加到文件末尾，文件指针始终在末尾。	ofstream, fstream
std::ios::ate	打开文件后，文件指针定位到文件末尾。但允许在文件任意位置读写。	所有
std::ios::trunc	如果文件已存在，将其内容截断为零长度（清空）。通常与 out 配合使用。	ofstream, fstream
std::ios::binary	以二进制模式打开文件（默认是文本模式）。	所有
注意：ifstream 的默认模式是 std::ios::in；ofstream 的默认模式是 std::ios::out；fstream 的默认模式是 std::ios::in | std::ios::out。

总结：
文件操作是c++中重要的功能，可以通过文件流来实现对文件的读写操作。使用文件流时需要注意打开文件、
进行读写操作和关闭文件的步骤，同时也要做好异常处理来保证程序的健壮性。
通过文件操作可以实现数据的持久化存储，使得程序产生的数据能够在程序结束后仍然存在。

读取文件操作：
getline(test, content);` 是 C++ 中用于从输入流（如文件流、标准输入等）中**读取一行文本**的函数。下面从函数原型、参数含义、工作原理、常见用法、注意事项以及与 `>>` 的区别等方面详细解释。
---
## 1. 函数原型
getline` 有两个常见版本，一个在 `<string>` 头文件中定义（用于 `std::string`），另一个是 `istream` 类的成员函数（用于 C 风格字符串）。我们最常用的是 **全局函数** 版本：
#include <string>
std::istream& getline(std::istream& is, std::string& str);
std::istream& getline(std::istream& is, std::string& str, char delim);
- **`is`**：输入流对象，例如 `std::cin`、`std::ifstream` 对象。
- **`str`**：一个 `std::string` 引用，用于存放读取到的内容（不包含换行符）。
- **`delim`**（可选）：分隔符，默认为 `'\n'`（换行符）。函数会一直读取直到遇到该分隔符，然后停止，分隔符本身被丢弃。
---
## 2. 工作原理
getline` 从输入流中逐个读取字符，直到：
- 遇到 **分隔符**（默认是换行符 `\n`）；
- 或遇到文件结束（EOF）；
- 或发生错误。
读取的字符（**不包括分隔符**）存入 `str` 中。**分隔符本身不会被存入字符串**，而是被从流中移除。如果遇到文件结束，则设置流的状态标志（`eofbit`）。
### 示例：
假设文件 `1.txt` 内容为：
Hello, World!
This is line 2.
执行 `getline(test, content);` 后，`content` 变成 `"Hello, World!"`，文件指针移动到第二行开头。再次调用 `getline(test, content);`，`content` 变成 `"This is line 2."`。
---
## 3. 返回值
getline` 返回输入流对象的引用（即第一个参数 `is`）。这个返回值可以用于**链式调用**或**条件判断**。
常见用法：
```cpp
if (getline(test, content)) {
    // 读取成功，可以使用 content
} else {
    // 读取失败（可能文件结束或错误）
}
```
因为流对象可以被隐式转换为 `bool`，所以可以直接在循环条件中使用：
while (getline(test, content)) {
    std::cout << content << std::endl;
}
当读取失败（如到达文件末尾）时，循环结束。
---
## 4. 与 `>>` 运算符的区别
| 特性 | `>>` 运算符 | `getline` |
|------|-------------|-----------|
| 读取内容 | 以空白字符（空格、制表符、换行）分隔，读取一个“单词” | 读取一整行，直到遇到换行符（或指定分隔符） |
| 分隔符处理 | 分隔符留在流中，下次读取会跳过 | 分隔符被读取并丢弃，不存入字符串 |
| 适用场景 | 读取以空白分隔的单词或数字 | 读取包含空格的一整行文本 |
| 示例 | `test >> word;`（读取直到空格） | `getline(test, line);`（读取整行） |
**重要**：如果之前使用 `>>` 读取后立即使用 `getline`，需要注意 `>>` 会留下换行符在流中，导致 `getline` 读到空行。解决办法是使用 `ignore()` 忽略残留的换行符。
test >> number;
test.ignore();          // 忽略换行符
getline(test, line);    // 正确读取下一行
---
## 5. 实际使用示例
### 5.1 从文件逐行读取并输出到屏幕
#include <iostream>
#include <fstream>
#include <string>
int main() {
    std::ifstream test("1.txt");
    if (!test.is_open()) {
        std::cerr << "无法打开文件\n";
        return 1;
    }

    std::string line;
    while (getline(test, line)) {
        std::cout << line << std::endl;
    }

    test.close();
    return 0;
}
### 5.2 读取时自定义分隔符（例如读取以逗号分隔的数据）
#include <iostream>
#include <fstream>
#include <string>
#include <sstream>
int main() {
    std::ifstream file("data.csv");
    std::string field;
    while (getline(file, field, ',')) {
        std::cout << "字段: " << field << std::endl;
    }
    return 0;
}
```
> 注意：如果行末没有逗号，最后一个字段读取后会正常结束；若混合换行和逗号，可能需要更复杂的解析。
---
## 6. 错误处理与状态标志
getline` 可能因以下原因失败：
- 流已处于 `eofbit`（文件结束）
- 流已处于 `failbit`（如遇到不可恢复的错误）
- 读取过程中发生异常（如果流设置了 `exceptions`）
成功读取后，流的状态 `goodbit` 为真。如果读取失败（如立即遇到 EOF），则 `str` 内容保持不变或为空（取决于实现），且流的 `failbit` 或 `eofbit` 被设置。
---
## 7. 注意事项
- **效率**：`getline` 会动态扩展 `std::string` 以容纳数据，无需担心缓冲区溢出。
- **空行**：如果文件有一行是空的（只有换行符），`getline` 会读取空字符串并成功（`str.empty()` 为 `true`）。
- **跨平台**：换行符在不同系统有差异（`\n` 在 Linux/macOS，`\r\n` 在 Windows），但 `getline` 会自动处理（对于文本模式打开的文件，流会将 `\r\n` 转换为 `\n`，`getline` 以 `\n` 作为分隔符）。
- **二进制文件**：如果以二进制模式打开文件，`getline` 仍能工作，但换行符不会转换，可能会读到 `\r` 字符。
---
## 8. 总结
getline(test, content);` 是 C++ 中读取文件一行的最常用方法，尤其适合处理包含空格的文本。它简单安全，结合循环可以轻松遍历文件的所有行。理解它与 `>>` 的区别，并注意混合使用时的换行符残留问题，能帮助你写出更健壮的文件读取代码。
好的，我来详细解释 `getline(test, content);` 中的两个参数。
---
## 1. 函数签名回顾
std::istream& getline(std::istream& is, std::string& str);
这是一个全局函数（定义在 `<string>` 头文件中），作用是从输入流中读取一行字符，存入字符串中。
---
## 2. 第一个参数：`test`
getline(test, content);
- **类型**：`std::istream&`（输入流引用）
- **含义**：表示你要从哪个输入流中读取数据。
- **为什么可以传 `test`**：`test` 是 `std::ifstream` 类型的对象，而 `std::ifstream` 是 `std::istream` 的派生类，所以可以隐式转换为 `std::istream&` 引用。
- **实际作用**：`getline` 函数会从这个流对象中逐个读取字符，直到遇到换行符（或文件结束）。这个流对象必须已经打开（例如通过 `open("1.txt")` 或构造函数），否则读取会失败。
**常见用法**：
- 从文件读取：`getline(ifstream对象, 字符串)`
- 从标准输入读取：`getline(std::cin, 字符串)`
- 从字符串流读取：`getline(stringstream对象, 字符串)`
**关键点**：函数接收的是引用，所以传入的是流对象本身（而不是副本），函数内部会直接操作这个流（改变其读取位置、状态标志等）。
---
## 3. 第二个参数：`content`
getline(test, content);
- **类型**：`std::string&`（字符串引用）
- **含义**：用于存放读取结果的字符串对象。
- **为什么用引用**：因为函数需要将读取到的字符存入这个字符串中。如果传值，函数内部修改的只是副本，外部无法获取结果。使用引用可以避免拷贝，并直接修改原对象。
- **实际作用**：`getline` 会将读取到的字符（不包括换行符）追加到 `content` 中（实际上是先清空原内容，再存入新内容）。调用前 `content` 可以是空字符串，也可以是已有内容的字符串，函数会覆盖它。
**示例**：
std::string line;
getline(test, line);   // 读取一行存入 line
**关键点**：第二个参数必须是 `std::string` 类型的左值（可修改的对象），不能是常量或临时字符串。
---
## 4. 调用后的变化
执行 `getline(test, content);` 后：
- **`test` 流对象**：读取指针移动到下一行开头（或文件末尾），状态标志可能改变（如果遇到 EOF，会设置 `eofbit`）。
- **`content` 字符串**：内容被替换为刚刚读取的一行文本（不包括换行符）。如果文件为空或立即遇到 EOF，`content` 可能变为空字符串。
---
## 5. 为什么这样设计？
C++ 的 I/O 流库设计遵循“流 + 缓冲区”模式，`getline` 作为一个独立函数，使得任何输入流（不限于文件流）都能方便地读取一行。使用引用传递允许函数直接修改流状态和字符串，符合 C++ 的效率和灵活性原则。
---
## 6. 与其他形式对比
- **成员函数版本**：`istream` 类也有一个成员函数 `getline(char* buffer, size_t size)`，但它针对 C 风格字符串，需要手动管理缓冲区，容易出错。全局的 `std::getline` 与 `std::string` 配合更安全。
- **`>>` 运算符**：`test >> content` 只能读取到空白字符，不能读取包含空格的整行。`getline` 弥补了这个不足。
---
## 7. 常见错误
- **忘记包含头文件**：`#include <string>` 是必须的，否则 `getline` 无法识别 `std::string` 版本。
- **混合使用 `>>` 和 `getline`**：如先 `test >> num` 再 `getline(test, line)`，会因 `>>` 留下的换行符导致 `getline` 读到空行。解决方法是在 `>>` 后调用 `test.ignore()` 忽略换行符。
- **在二进制模式下使用**：如果以二进制模式打开文件，`getline` 仍然可以工作，但换行符不会自动转换（Windows 下可能出现 `\r\n` 被当作两个字符），可能导致行内容包含 `\r`。
---
## 8. 总结
- **第一个参数**：输入流对象，指定数据来源（如文件流、标准输入）。
- **第二个参数**：字符串引用，用于存放读取结果，函数直接修改它。
- **函数作用**：从输入流中读取一行（直到换行符），丢弃换行符，将内容存入字符串。
理解这两个参数的类型和意义，就能正确使用 `getline` 来读取文件或用户输入的行数据。

在 C++ 中，向文件写入数据主要通过**输出文件流对象**（`std::ofstream` 或 `std::fstream`）的以下方式实现：
---
## 1. 最常用的写入方式：插入运算符 `<<`
std::ofstream outFile("output.txt");
outFile << "Hello, World!" << std::endl;
outFile << 123 << " " << 45.67;
```
- **作用**：将各种类型的数据（字符串、整数、浮点数等）以**文本形式**写入文件。
- **特点**：自动进行格式化转换，使用方便，适合文本文件。
---
## 2. 二进制写入：`write()` 成员函数
std::ofstream outFile("data.bin", std::ios::binary);
int arr[] = {1, 2, 3};
outFile.write(reinterpret_cast<char*>(arr), sizeof(arr));
- **原型**：`std::ostream& write(const char* buffer, std::streamsize size);`
- **作用**：将内存中的原始字节序列原样写入文件，不进行任何转换。
- **适用**：二进制文件、结构体、对象序列化等。
---
## 3. 其他写入成员函数
| 函数 | 说明 |
|------|------|
| `put(char c)` | 写入单个字符 |
| `write(const char* s, streamsize n)` | 写入指定长度的字符序列 |
| `flush()` | 强制刷新缓冲区，确保数据真正写入磁盘 |

## 6. 与读取的对应关系
| 操作方向 | 文本模式 | 二进制模式 |
|----------|----------|------------|
| 写入 | `<<` 运算符 | `write()` |
| 读取 | `>>` 运算符 或 `getline()` | `read()` |
---
## 7. 注意事项
- **打开模式**：`std::ofstream` 默认以 `std::ios::out` 打开（清空原有内容）。若需追加，使用 `std::ios::app`。
- **错误检查**：写入前应检查文件是否成功打开，写入后可检查流状态。
- **缓冲区**：写入的数据可能暂存在缓冲区，关闭文件或调用 `flush()` 才能确保写入磁盘。
如果需要写入特定格式的数据，`<<` 是最直接的选择；如果需要写入非文本数据（如结构体、数组），应使用 `write()` 配合二进制模式。

文件类型分为：
文本文件：以文本形式存储ASCII码存储数据，可以使用文本编辑器打开和编辑，常见的文本文件扩展名有.txt、.csv、.log等
二进制文件：以二进制形式存储数据，也就是01码，不能直接使用文本编辑器打开和编辑，常见的二进制文件扩展名有.dat、.bin、.exe等
文本文件和二进制文件的区别：
文本文件以可读的文本形式存储数据，可以直接使用文本编辑器打开和编辑，适合存储人类可读的数据；二进制文件以二进制形式存储数据，不能直接使用文本编辑器打开和编辑，适合存储需要高效读取和写入的数据。
文本文件的读写操作：
使用输入文件流对象的成员函数进行读取操作，例如：inFile >> data; inFile.getline(buffer, size);
使用输出文件流对象的成员函数进行写入操作，例如：outFile << data; outFile.write(buffer, size);
二进制文件的读写操作：
使用输入文件流对象的成员函数进行读取操作，例如：inFile.read(buffer, size);
使用输出文件流对象的成员函数进行写入操作，例如：outFile.write(buffer, size);
在打开文件时需要指定打开模式，例如：inFile.open("input.dat", std::ios::in | std::ios::binary); outFile.open("output.dat", std::ios::out | std::ios::binary);
总结：
文本文件和二进制文件是两种不同类型的文件，文本文件以可读的文本形式存储数据，适合存储人类可读的数据；
二进制文件以二进制形式存储数据，适合存储需要高效读取和写入的数据。
在进行文件操作时，需要根据文件类型选择合适的读写方式，并且在打开文件时指定正确的打开模式.

5.1 文本文件

5.1.1 文本文件的写操作
文本文件的写操作主要通过输出文件流对象（`std::ofstream`）的插入运算符 `<<` 来实现，可以将各种类型的数据以文本形式写入文件中。
例如：
1.包含头文件
#include <fstream>
2.创建输出文件流对象
std::ofstream outFile；
3.打开文件
outFile.open("文件路径",打开方式);
4.写入数据
outFile << "要写入的数据";
5.关闭文件
outFile.close();

文件打开方式：
打开方式	说明
std::ios::in	以读取方式打开文件，如果文件不存在，打开失败.
std::ios::out	以写入方式打开文件，如果文件已存在，默认会清空文件原有内容。
std::ios::app	以追加方式写入，所有输出都添加到文件末尾，文件指针始终在末尾。
std::ios::ate	打开文件后，文件指针定位到文件末尾，但允许在文件任意位置读写。
std::ios::trunc	如果文件已存在，将其内容截断为零长度（清空）。通常与 out 配合使用。
std::ios::binary	以二进制模式打开文件（默认是文本模式）.
